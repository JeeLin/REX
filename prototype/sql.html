分析当前查询，发现以下性能问题：

1. 全表扫描
LEFT JOIN orders 在 users 表上，但没有利用索引。建议： 
CREATE INDEX idx_orders_user_id ON orders(user_id);
2. HAVING 子句中的聚合过滤
HAVING COUNT(o.id) > 5 会在分组后才过滤，建议改用子查询： 
WHERE u.id IN (
  SELECT user_id FROM orders
  WHERE created_at >= '2024-01-01'
  GROUP BY user_id HAVING COUNT(*) > 5
)
⚠ 以上建议基于表结构分析，未考虑实际数据分布。执行 EXPLAIN 验证执行计划后再决定是否采纳。
