## 子任务详细设计

### 27.1 实现 rex-mysql 真实连接器 + WebSocket handler

**功能目标：**
使用 `sqlx` crate 实现真实的 MySQL 连接，替换当前的 stub 实现。创建 WebSocket handler 使前端 SQL 控制台能实际执行 MySQL 查询。

**修改/创建文件：**
```text
Cargo.toml                          修改：添加 sqlx workspace 依赖
crates/rex-mysql/Cargo.toml         修改：添加 sqlx、tokio 依赖
crates/rex-mysql/src/connector.rs   修改：实现真实的 connect/execute/list_* 方法
crates/rex-hub/src/ws_mysql.rs      新增：MySQL WebSocket handler
crates/rex-hub/src/routes.rs        修改：注册 /ws/mysql/:resource_id 路由
```

**依赖选型：**

`sqlx`（非阻塞，支持 MySQL + 连接池，与 tokio 生态一致）：
```toml
# 根 Cargo.toml [workspace.dependencies]
sqlx = { version = "0.8", features = ["runtime-tokio", "tls-rustls", "mysql"] }
```

**connector.rs 实现要点：**

```rust
use sqlx::mysql::{MySqlPool, MySqlPoolOptions};

pub struct MySqlConnector {
    config: MySqlConfig,
    pool: Option<MySqlPool>,
}

// connect(): 建立连接池
// execute(): sqlx::query_as + 映射为 SqlResult
// list_databases(): SHOW DATABASES
// list_tables(): SHOW TABLES FROM {database}
// list_columns(): SHOW COLUMNS FROM {table} FROM {database}
// close(): pool.close()
```

**ws_mysql.rs 设计：**

参照 `ws_sqlite.rs` 的架构，使用相同的消息协议：
```text
ClientMsg: Command { id, action, params } | Ping
ServerMsg: Response { id, data } | Error { id, message } | Pong | Connected { server } | Disconnected { reason }
```

action 分发：
- `execute` → connector.execute(sql)
- `databases` → connector.list_databases()
- `tables` → connector.list_tables(database)
- `columns` → connector.list_columns(database, table)

**连接管理：**
- 连接池在 WebSocket 生命周期内保持
- WebSocket 关闭时 pool.close()
- 超时或错误时发送 Disconnected 消息

**routes.rs 路由注册：**
```rust
.route("/ws/mysql/:resource_id", get(crate::ws_mysql::mysql_ws_handler))
```

**测试标准：**
- connector 连接/断开逻辑测试（mock 或真实 MySQL）
- execute 返回正确的 SqlResult 格式
- list_databases/tables/columns 返回正确数据
- WebSocket 消息序列化/反序列化
- 连接失败时返回错误消息

**提交信息：**
```
feat: implement real MySQL connection with sqlx
```

### 27.2 实现 rex-postgresql 真实连接器 + WebSocket handler

**功能目标：**
使用 `deadpool-postgres`（或 `sqlx` 的 postgres feature）实现真实的 PostgreSQL 连接。

**修改/创建文件：**
```text
Cargo.toml                              修改：添加 postgres 依赖
crates/rex-postgresql/Cargo.toml        修改：添加依赖
crates/rex-postgresql/src/connector.rs  修改：实现真实连接方法
crates/rex-hub/src/ws_postgresql.rs     新增：PostgreSQL WebSocket handler
crates/rex-hub/src/routes.rs            修改：注册 /ws/postgresql/:resource_id 路由
```

**依赖选型（二选一）：**

方案 A — `sqlx` 统一（推荐，与 MySQL 一致）：
```toml
sqlx = { version = "0.8", features = ["runtime-tokio", "tls-rustls", "mysql", "postgres"] }
```

方案 B — `deadpool-postgres`（独立连接池）：
```toml
deadpool-postgres = "0.14"
tokio-postgres = { version = "0.7", features = ["with-serde_json-1"] }
```

推荐方案 A：统一使用 `sqlx`，减少依赖碎片化。

**connector.rs 实现要点：**
```rust
use sqlx::postgres::{PgPool, PgPoolOptions};

pub struct PostgresConnector {
    config: PostgresConfig,
    pool: Option<PgPool>,
}

// connect(): 建立连接池
// execute(): sqlx::query_as + 映射为 SqlResult
// list_databases(): SELECT datname FROM pg_database WHERE datistemplate = false
// list_tables(): SELECT tablename FROM pg_tables WHERE schemaname = 'public'
// list_columns(): SELECT column_name, data_type, ... FROM information_schema.columns WHERE table_name = $1
// close(): pool.close()
```

**ws_postgresql.rs 设计：**

与 ws_mysql.rs 相同的消息协议和 action 分发，区别在于：
- `databases` → 连接到 `postgres` 系统数据库列出所有数据库
- `tables` → 查 `pg_tables`
- `columns` → 查 `information_schema.columns`

**routes.rs 路由注册：**
```rust
.route("/ws/postgresql/:resource_id", get(crate::ws_postgresql::postgresql_ws_handler))
```

**测试标准：**
- 连接/断开逻辑测试
- execute 返回正确格式
- list_databases/tables/columns 查询正确
- WebSocket 消息协议测试

**提交信息：**
```
feat: implement real PostgreSQL connection with sqlx
```
