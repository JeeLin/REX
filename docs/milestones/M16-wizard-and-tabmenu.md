# M16: 资源创建测试连接 + 部署指南改进 + 移动端修复

## Context

M14 完成了工作区面板集成，M15 改进了 Agent 部署指南。当前存在三个问题：
1. 资源创建向导缺少测试连接功能
2. Agent 部署指南二进制下载仍有 curl 命令且无移动端适配
3. 移动端侧边栏点击后主内容不可交互

## 产品边界

**做什么：**
- 资源创建向导增加测试连接按钮（SSH/SFTP/MySQL/PostgreSQL）
- Agent 部署指南二进制下载去掉 curl 命令，保留直接下载按钮
- Agent 部署指南增加移动端 CSS 适配
- 修复移动端侧边栏 z-index 导致主内容不可点击的问题

**不做什么：**
- 不实现标签右键菜单"移动到面板"（移至 M17）
- 不实现多协议表单（目前只支持 SSH，其他协议后续里程碑）
- 不实现面板间拖拽（属于后续里程碑）

## 子任务详细设计

### 16.1 资源创建向导 4 步重构

**功能目标：**
将资源创建向导从 3 步扩展为 4 步，恢复 PRODUCT.md 定义的完整流程。

**修改文件：**
```text
packages/rex-console-web/src/
├── pages/ResourceNew.vue                    修改：4 步向导
├── i18n/zh.ts, en.ts                        修改：添加向导 i18n
```

**接口设计：**

```ts
// ResourceNew.vue 数据模型
const form = reactive({
  protocol: '',
  name: '',
  envId: '',           // 新增：所属环境 ID
  connectionMode: 'agent' as 'agent' | 'direct',  // 新增：连接方式
  config_json: '',
})
```

**交互设计（参考 prototype/resource-new.html）：**

**步骤指示器（4 步）：**
```text
(1) 选择协议 ── (2) 基本信息 ── (3) 连接详情 ── (4) 完成
```

**第 1 步：选择协议**（保持不变）
- 8 个协议卡片网格
- 选中后高亮边框，自动进入第 2 步

**第 2 步：基本信息 + 连接方式**（新增环境选择和连接方式）
```text
┌─────────────────────────────────────────────────┐
│ 资源名称                                         │
│ [输入框 placeholder: 例如：云服务器、主数据库]        │
│                                                   │
│ 所属环境                                         │
│ [下拉框: 已有环境列表 + "+ 创建新环境"]              │
│                                                   │
│ 连接方式                                         │
│ ┌──────────────────┐  ┌──────────────────┐       │
│ │ ⬡ Agent 代理      │  │ ◉ 直连            │       │
│ │ 通过内网 Agent... │  │ Hub 直接连接...   │       │
│ └──────────────────┘  └──────────────────┘       │
└─────────────────────────────────────────────────┘
```

- 环境下拉框：从 API 获取环境列表，末尾追加"+ 创建新环境"选项
- 选择"+ 创建新环境"时，弹出简化的创建对话框（名称输入），创建后自动选中
- 连接方式：两张卡片并排，默认选中"Agent 代理"
- 选中 Agent 代理时显示提示文字：「部署 Agent 后，此环境下的所有资源将通过 Agent 代理访问」
- 选中直连时提示：「REX Hub 服务器直接连接目标资源」

**第 3 步：连接详情**（新增测试连接按钮）
- 保持现有 SSH 表单（主机、端口、用户名、认证方式）
- 底部增加「测试连接」按钮
- 测试连接状态：未测试 / 测试中 / 成功（绿色提示）/ 失败（红色提示）
- 测试连接调用 `/api/resources/test`（新增 API 端点）

**第 4 步：完成**（新增"立即连接"按钮）
```text
┌─────────────────────────────────────┐
│              ✓                       │
│     资源创建成功                      │
│  资源已保存到 [环境名] 环境            │
│                                      │
│  [立即连接]  [返回环境详情]            │
└─────────────────────────────────────┘
```

**后端流程：**

新增测试连接 API：
```text
POST /api/resources/test
Body: { protocol, config_json }
Response: { success: boolean, message: string, latency_ms?: number }
```

后端根据 protocol 和 config_json 创建临时连接，测试连通性后关闭。

**测试标准：**
- 4 步步骤指示器正确切换
- 环境下拉框显示所有环境
- 选择"+ 创建新环境"能创建并自动选中
- 连接方式选择正确切换提示文字
- 测试连接按钮调用 API 并显示结果
- 完成页显示"立即连接"按钮
- 环境详情页的"添加资源"按钮正确传递 envId

**提交信息：**
```
feat: expand resource creation wizard to 4 steps with env selector and connection test
```

### 16.2 标签右键菜单增加"移动到面板"

**功能目标：**
在标签右键菜单中增加"移动到面板"子菜单，允许用户在分屏模式下将标签移动到指定面板。

**修改文件：**
```text
packages/rex-console-web/src/
├── features/workspace/TabBar.vue          修改：增加菜单项
├── features/workspace/useTabs.ts          修改：增加 moveToPanel 方法
├── i18n/zh.ts, en.ts                      修改：添加 i18n
```

**交互设计（参考 prototype/app.html 标签右键菜单）：**

```text
关闭
关闭其他
关闭右侧
关闭左侧
关闭全部
─── 分隔线 ───
复制标签
移动到面板 ▸  面板 1 / 面板 2 / 面板 3 / 面板 4
─── 分隔线 ───
新建连接
全部断开
```

**移动到面板子菜单：**
- 仅在分屏模式（panelCount > 1）下显示
- 列出所有面板编号，当前所在面板灰显
- 点击后面板编号切换，标签内容移动到目标面板

**useTabs.ts 接口：**
```ts
function moveToPanel(tabId: string, panelIndex: number): void
```

**测试标准：**
- 单面板模式下不显示"移动到面板"菜单项
- 分屏模式下显示所有面板编号
- 当前面板灰显不可选
- 移动后面板内容正确切换
- 全部断开菜单项正确触发

**提交信息：**
```
feat: add move-to-panel submenu in workspace tab context menu
```

### 16.3 Agent 部署指南二进制下载改进 + 移动端适配

**功能目标：**
1. 二进制下载 Tab 改为直接提供下载按钮（从 Hub 服务下载），去掉 curl 命令代码块
2. 部署指南整体增加移动端适配

**修改文件：**
```text
packages/rex-console-web/src/
├── features/agents/DeployGuide.vue         修改：下载按钮 + 移动端样式
├── i18n/zh.ts, en.ts                       修改：调整 i18n
```

**交互设计：**

**二进制下载 Tab 改进：**

去掉 curl 命令后：
```text
步骤 1：下载 Agent
┌──────────────────────────────────────────┐
│ 选择目标系统对应的二进制文件                │
│                                          │
│ ⬇ linux-amd64   ⬇ linux-arm64            │
│ ⬇ linux-armv7l  ⬇ darwin-arm64           │
│ ⬇ darwin-amd64  ⬇ windows-amd64          │
└──────────────────────────────────────────┘
```

- 保留 6 个下载按钮，链接到 `/api/agent/download?os={os}&arch={arch}`
- 去掉 curl 命令代码块及其复制按钮
- 每个下载按钮显示平台名称 + 架构

**移动端适配：**

```text
桌面端（>767px）：
┌──────────────────────────────────────────┐
│ ⬇ linux-amd64  ⬇ linux-arm64  ⬇ armv7l │
│ ⬇ darwin-arm64 ⬇ darwin-amd64 ⬇ win64  │
└──────────────────────────────────────────┘

移动端（≤767px）：
┌────────────────────────┐
│ ⬇ linux-amd64          │
│ ⬇ linux-arm64          │
│ ⬇ linux-armv7l         │
│ ⬇ darwin-arm64         │
│ ⬇ darwin-amd64         │
│ ⬇ windows-amd64        │
└────────────────────────┘
```

- 下载按钮网格在移动端切换为单列布局
- 步骤编号和内容区在小屏幕上纵向排列
- 标签栏在移动端可横向滚动
- 令牌框在移动端正确截断显示

**测试标准：**
- 6 个下载按钮链接到正确的 API 路径
- 点击下载按钮触发浏览器下载
- curl 命令代码块已移除
- 移动端（≤767px）下载按钮单列显示
- 移动端标签栏可横向滚动
- 移动端令牌框正确截断

