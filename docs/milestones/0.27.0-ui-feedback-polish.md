# 0.27.0: UI 一致性与交互反馈

## Context

0.26.0 完成了全局搜索命令面板。0.24.0 修复了 16 项 Bug 与体验改进，0.25.0 提升了测试覆盖率，0.26.0 补齐了核心交互入口。本里程碑聚焦 UI 层的精细化——统一加载态、错误态、空态的视觉和交互模式，提升操作反馈的即时性，使整体体验更加专业一致。

版本类型：minor（新增功能，向后兼容），版本号 0.27.0。

## 产品边界

**做什么：**
- 统一所有页面的三态处理（加载中 / 错误 / 空状态）
- 补充缺失的 Toast 反馈（操作成功/失败提示）
- 补充缺失的确认弹窗（危险操作二次确认）
- 改进表格组件：排序、筛选、分页交互
- 统一移动端适配（响应式断点、触控交互）

**不做什么：**
- 不新增后端 API
- 不修改核心业务逻辑
- 不引入新协议或新资源类型
- 不改变产品定位和架构

## 子任务详细设计

### 1 统一三态组件

- **功能目标**：创建通用的 LoadingSpinner、ErrorState、EmptyState 组件，在所有页面统一使用。
- **文件结构**：
  - 创建：`packages/rex-console-web/src/components/LoadingSpinner.vue`
  - 创建：`packages/rex-console-web/src/components/ErrorState.vue`
  - 创建：`packages/rex-console-web/src/components/EmptyState.vue`
  - 修改：所有页面（Dashboard、Environments、EnvironmentDetail、Agents、AuditLog、Settings）
- **接口设计**：
  - LoadingSpinner：`size?: 'sm' | 'md' | 'lg'`，`text?: string`
  - ErrorState：`message: string`，`retry?: () => void`
  - EmptyState：`icon?: string`，`title: string`，`hint?: string`，`action?: { label: string, handler: () => void }`
- **交互设计**：
  - LoadingSpinner：旋转动画，橙色主题色，3 种尺寸
  - ErrorState：错误图标 + 错误信息 + 重试按钮（可选）
  - EmptyState：淡色图标 + 标题 + 说明文字 + 操作按钮（可选）
- **测试标准**：组件渲染正确，props 生效。
- **提交信息**：`feat: add unified LoadingSpinner, ErrorState, EmptyState components`

### 2 补充 Toast 反馈与确认弹窗

- **功能目标**：确保所有 CRUD 操作都有 Toast 反馈，危险操作有确认弹窗。
- **文件结构**：
  - 修改：`packages/rex-console-web/src/composables/useToast.ts`（如需扩展）
  - 修改：`packages/rex-console-web/src/components/ConfirmDialog.vue`（如需创建）
  - 修改：各页面（添加 Toast 调用）
- **接口设计**：
  - Toast：`success(message)` / `error(message)` / `warning(message)` / `info(message)`
  - ConfirmDialog：`title`、`message`、`confirmLabel`、`onConfirm`、`onCancel`
- **交互设计**：
  - Toast：右上角滑入，3 秒自动消失，手动关闭按钮
  - ConfirmDialog：模态弹窗，半透明遮罩，红色确认按钮（危险操作）
- **测试标准**：CRUD 操作后 Toast 正确显示，删除操作弹出确认框。
- **提交信息**：`feat: add Toast feedback and ConfirmDialog for all CRUD operations`

### 3 表格组件增强

- **功能目标**：统一表格组件，支持排序、筛选、分页。
- **文件结构**：
  - 创建：`packages/rex-console-web/src/components/DataTable.vue`
  - 修改：Environments、Agents、AuditLog 页面（替换为 DataTable）
- **接口设计**：
  - Props：`columns`、`data`、`sortable`、`filterable`、`pagination`
  - Events：`sort`、`filter`、`page-change`
- **交互设计**：
  - 列头点击排序（升序/降序/默认）
  - 筛选输入框（实时过滤）
  - 分页控件（上一页/下一页/页码）
- **测试标准**：排序、筛选、分页功能正常。
- **提交信息**：`feat: add DataTable component with sorting, filtering, pagination`

### 4 移动端响应式适配改进

- **功能目标**：确保所有页面在移动端（<768px）正常显示和交互。
- **文件结构**：
  - 修改：各页面（添加响应式样式）
  - 修改：`packages/rex-console-web/src/styles/responsive.css`（如需创建）
- **接口设计**：
  - 断点：`mobile: <768px`，`tablet: 768-1024px`，`desktop: >1024px`
  - 移动端：表格转卡片、侧边栏隐藏、底部导航
- **交互设计**：
  - 移动端表格：每行转为卡片布局，隐藏次要列
  - 移动端导航：底部固定导航栏（仪表盘、环境、新建、Agent、设置）
  - 触控优化：按钮最小 44px、间距增大
- **测试标准**：在 375px（iPhone SE）和 768px（iPad）下布局正常。
- **提交信息**：`feat: improve mobile responsive layout for all pages`
