# 0.28.0: Docker 真实连接实现

## Context

0.27.2 完成了 HTTPS/TLS 修复。0.9.0 曾实现 Docker 协议支持，但所有 connector 方法均为 stub（返回空数据），实际无法管理容器。当前 Docker 相关代码已从工作区回滚（crate、Hub handler、前端全部删除），需要以真实连接为目标重新实现。

Docker Engine API 基于 HTTP，Hub 可直连本地 Docker daemon（Unix socket）或远程 Docker daemon（TCP）。Agent 代理路径的隧道协议尚未实现，本里程碑仅支持直连。

版本类型：minor（恢复并增强已有功能，从 stub 升级为真实连接）

## 产品边界

**做什么：**
- 恢复 `rex-docker` crate，实现真实 Docker Engine API 通信（基于 `reqwest`）
- 支持 Unix socket（`/var/run/docker.sock`）和 TCP（`tcp://host:2375`）两种连接方式
- 实现容器生命周期操作：列表、详情、启动、停止、重启、删除、日志
- 恢复 Hub Docker WebSocket handler（`ws_docker.rs`）
- 恢复前端 Docker 管理控制台（容器列表、日志、状态操作）
- 恢复 Docker 资源创建向导表单和工作空间面板集成
- Docker 资源配置验证（`host` 字段格式校验）

**不做什么：**
- 不实现 Agent 代理路径（隧道协议另立里程碑）
- 不实现 Docker Compose 集成
- 不实现镜像管理、网络管理、卷管理
- 不实现容器内文件浏览（通过 SSH/SFTP 替代）
- 不实现 Docker Swarm 集群管理

## 子任务详细设计

### 1 恢复 rex-docker crate + 实现真实 Docker Engine API 连接

- **功能目标**
  恢复 `rex-docker` crate，将 stub 实现替换为真实的 Docker Engine API HTTP 客户端。使用 `reqwest`（已是 workspace 依赖）直接调用 Docker Engine REST API，支持 Unix socket 和 TCP 两种传输方式。

- **文件结构**（创建/修改哪些文件）
  ```text
  crates/rex-docker/Cargo.toml         创建：恢复 crate 配置
  crates/rex-docker/src/lib.rs         创建：导出模块
  crates/rex-docker/src/connector.rs   创建：Docker 连接器（真实实现）
  ```

- **接口设计**

  数据模型（与 0.9.0 stub 保持一致，前端已适配）：
  ```rust
  // DockerConfig — 连接配置
  pub struct DockerConfig {
      pub host: String,       // "unix:///var/run/docker.sock" 或 "tcp://host:port"
      pub name: Option<String>,
  }

  // ContainerState — 容器状态枚举
  enum ContainerState { Running, Paused, Stopped, Created, Dead }

  // PortMapping — 端口映射
  struct PortMapping { private: u16, public: Option<u16>, protocol: String }

  // ContainerInfo — 容器信息（列表返回）
  struct ContainerInfo { id, name, image, state, status, created, ports }
  ```

  DockerConnector trait（与 0.9.0 一致）：
  ```rust
  #[async_trait]
  pub trait DockerConnector: Send + Sync {
      async fn connect(&mut self) -> Result<()>;
      async fn list_containers(&self, all: bool) -> Result<Vec<ContainerInfo>>;
      async fn inspect_container(&self, id: &str) -> Result<serde_json::Value>;
      async fn start_container(&self, id: &str) -> Result<()>;
      async fn stop_container(&self, id: &str) -> Result<()>;
      async fn restart_container(&self, id: &str) -> Result<()>;
      async fn remove_container(&self, id: &str) -> Result<()>;
      async fn logs(&self, id: &str, tail: u32) -> Result<String>;
      async fn info(&self) -> Result<HashMap<String, String>>;
      async fn close(&self) -> Result<()>;
  }
  ```

- **后端流程**

  `DockerConnectorImpl` 真实实现：
  ```rust
  pub struct DockerConnectorImpl {
      config: DockerConfig,
      client: Option<reqwest::Client>,
      base_url: String,  // "http://localhost" (unix) 或 "http://host:port" (tcp)
  }
  ```

  `connect()`:
  1. 根据 `config.host` 判断连接方式：
     - `unix://` 路径 → 使用 `reqwest::ClientBuilder` 配置 Unix socket transport（通过 `reqwest-unix-connector` 或 `tokio::net::UnixStream` 手动建立）
     - `tcp://` → 直接使用 `reqwest::Client`，base_url 为 `http://{host}:{port}`
  2. 发送 `GET /_ping` 验证连接
  3. 存储 client 和 base_url

  各操作方法映射到 Docker Engine API：
  | 方法 | HTTP 请求 |
  |------|-----------|
  | `list_containers(all)` | `GET /containers/json?all={all}` |
  | `inspect_container(id)` | `GET /containers/{id}/json` |
  | `start_container(id)` | `POST /containers/{id}/start` |
  | `stop_container(id)` | `POST /containers/{id}/stop` |
  | `restart_container(id)` | `POST /containers/{id}/restart` |
  | `remove_container(id)` | `DELETE /containers/{id}?force=true` |
  | `logs(id, tail)` | `GET /containers/{id}/logs?stdout=true&stderr=true&tail={tail}` |
  | `info()` | `GET /info` |

  响应解析：`list_containers` 解析 Docker API JSON 响应，映射 `State`/`Status`/`Ports` 到 `ContainerInfo` 结构。

  Unix socket 支持方案：
  ```toml
  # crates/rex-docker/Cargo.toml
  [dependencies]
  reqwest = { workspace = true }
  reqwest-unix-connector = "0.2"  # Unix socket transport for reqwest
  ```

  ```rust
  // connect() 中 Unix socket 的处理
  let client = if host.starts_with("unix://") {
      let socket_path = host.strip_prefix("unix://").unwrap();
      let connector = reqwest_unix_connector::UnixConnector::new(socket_path);
      reqwest::Client::builder()
          .connector_layer(tower::ServiceBuilder::new().layer_fn(|_| connector.clone()))
          .build()?
  } else {
      reqwest::Client::new()
  };
  ```

  备选方案：如果没有合适的 Unix socket connector crate，可以使用 `bollard` crate（成熟的 Docker API 客户端），但它会引入较多依赖。优先使用 `reqwest` + Unix connector 保持轻量。

### 2 恢复 Hub Docker WebSocket handler + 路由注册 + 资源验证

- **功能目标**
  恢复 Hub 侧的 Docker WebSocket 会话管理（`ws_docker.rs`），注册路由，恢复 Docker 协议在 `VALID_PROTOCOLS` 中的注册，恢复资源创建时的配置验证。

- **文件结构**（创建/修改哪些文件）
  ```text
  crates/rex-hub/src/ws_docker.rs    创建：Docker WebSocket handler
  crates/rex-hub/src/lib.rs          修改：添加 pub mod ws_docker
  crates/rex-hub/src/routes.rs       修改：添加 /ws/docker/:resource_id 路由
  crates/rex-hub/src/resource.rs     修改：VALID_PROTOCOLS 添加 "docker"，create_resource 添加 Docker 配置验证
  ```

- **接口设计**

  WebSocket 消息协议（与 0.9.0 一致，前端已适配）：
  ```rust
  // Client → Hub
  enum DockerClientMsg {
      Command { id: String, action: String, params: serde_json::Value },
      Ping,
  }

  // Hub → Client
  enum DockerServerMsg {
      Response { id: String, data: serde_json::Value },
      Error { id: String, message: String },
      Pong,
      Connected { server: HashMap<String, String> },
      Disconnected { reason: String },
  }
  ```

  action 分发：
  | action | 处理 |
  |--------|------|
  | `list` | `connector.list_containers(params.all)` |
  | `inspect` | `connector.inspect_container(params.id)` |
  | `start` | `connector.start_container(params.id)` |
  | `stop` | `connector.stop_container(params.id)` |
  | `restart` | `connector.restart_container(params.id)` |
  | `remove` | `connector.remove_container(params.id)` |
  | `logs` | `connector.logs(params.id, params.tail)` |

- **后端流程**

  `handle_docker_socket` 函数（参照 `ws_redis.rs` 模式）：
  1. 分离 WebSocket 为读/写半部
  2. 从 SQLite 读取 `config_json`
  3. 解析 `DockerConnectorImpl::from_json(&config_json)`
  4. 调用 `connector.connect().await` 建立 Docker API 连接
  5. 调用 `connector.info().await` 获取服务器信息
  6. 发送 `Connected { server }` 消息
  7. 进入消息循环：接收 Command → 调用 connector 方法 → 发送 Response
  8. 断开时清理

  `routes.rs` 路由注册：
  ```rust
  .route("/ws/docker/:resource_id", get(crate::ws_docker::docker_ws_handler))
  ```

  `resource.rs` 修改：
  - `VALID_PROTOCOLS` 添加 `"docker"`
  - `create_resource` 添加 Docker 配置验证（解析 `DockerConfig` 校验 `host` 字段格式）

- **测试标准**
  - WebSocket 消息序列化/反序列化测试
  - action 分发逻辑测试
  - `DockerConfig` 验证逻辑测试
  - 路由注册正确性
