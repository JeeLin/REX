# 0.7.0: Hub 自动证书（ACME + 自签名）

## Context

0.6.0 实现了 Hub TLS/HTTPS 支持，但需要用户手动放置证书文件。对于自托管用户，手动管理证书（申请、续期、替换）是痛点。本里程碑实现自动证书管理：优先使用 ACME（Let's Encrypt）自动申请，不支持 ACME 时自动降级为自签名证书。

版本类型：minor

## 产品边界

**做什么：**
- Hub 启动时自动选择 TLS 模式（按优先级）：
  1. **手动证书**（`tls.cert` + `tls.key`）→ 直接使用用户提供的证书
  2. **ACME 域名证书**（`acme.domain` 为域名）→ Let's Encrypt 申请，90 天有效期
  3. **ACME IP 证书**（`acme.domain` 为 IP 地址）→ Let's Encrypt 申请，6 天有效期
  4. **自签名证书**（无 ACME 配置）→ 自动生成，适用于内网
- 证书自动续期：域名证书到期前 30 天续期，IP 证书到期前 4 天续期
- 证书持久化到本地磁盘，重启后无需重新申请
- 支持 staging 环境测试（避免触发 Let's Encrypt rate limit）
- 前端设置页展示 TLS 状态和配置指引

**限制：**
- ACME 仅适用于**公网 IP 或域名**（Let's Encrypt 需要能访问验证端口）
- HTTP-01 challenge 需要 80 端口对外可达
- TLS-ALPN-01 challenge 需要 443 端口对外可达
- 自签名证书会被浏览器标记为不安全（需用户手动安装 CA 或接受警告）

**不做什么：**
- 不支持除 Let's Encrypt 以外的 ACME CA
- 不支持 dns-01 验证（仅支持 http-01 和 tls-alpn-01）
- 不实现证书吊销（短期证书，吊销意义不大）
- 不修改 0.6.0 的手动证书配置

## 子任务详细设计

### 1 ACME 客户端与自签名证书生成

- **功能目标**
  集成 ACME 客户端（Let's Encrypt）和自签名证书生成。支持域名、IP、内网三种场景。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-hub/src/acme.rs` — ACME 客户端封装
  - 新增：`crates/rex-hub/src/self_signed.rs` — 自签名证书生成
  - 修改：`crates/rex-hub/src/config.rs` — HubConfig 增加 acme 配置段
  - 修改：`crates/rex-common/src/cli.rs` — 增加 --acme-domain、--acme-email 参数
  - 修改：`crates/rex-hub/Cargo.toml` — 添加 acme + rcgen 依赖
  - 修改：`Cargo.toml`（workspace）— 添加 acme + rcgen workspace 依赖

- **接口设计**
  配置文件（hub.yaml）：
  ```yaml
  # 方式一：手动证书（最高优先级）
  tls:
    cert: "/path/to/cert.pem"
    key: "/path/to/key.pem"

  # 方式二：ACME 自动证书
  acme:
    domain: "hub.example.com"    # 域名或公网 IP
    email: "admin@example.com"   # Let's Encrypt 账户邮箱
    staging: false                # true = 使用 staging 环境

  # 方式三：自签名（无 acme 配置时自动启用）
  # 无需配置，自动检测
  ```

  环境变量：
  - `REX_ACME_DOMAIN=hub.example.com`
  - `REX_ACME_EMAIL=admin@example.com`
  - `REX_ACME_STAGING=false`

  CLI 参数：
  - `--acme-domain <domain>`
  - `--acme-email <email>`
  - `--acme-staging`

- **后端流程**
  1. 解析配置，确定 TLS 模式（优先级：manual > acme > self-signed）
  2. ACME 模式：
     a. 自动识别 domain 类型（域名 or IP）
     b. 域名 → HTTP-01 challenge（80 端口）
     c. IP → TLS-ALPN-01 challenge（443 端口）
     d. 完成 challenge → 获取证书
  3. 自签名模式：
     a. 使用 `rcgen` 生成 CA 证书和服务器证书
     b. 证书包含 Subject Alternative Names（IP 或 localhost）
     c. 保存到 `data/self-signed/` 目录

- **依赖选型**
  - `rcgen` — 自签名证书生成
  - `rustls-acme` 或 `acme-rs` — ACME 客户端
  - 已有：`rustls`、`tokio-rustls`、`rustls-pemfile`

- **测试标准**
  - 单元测试：配置解析（域名/IP/自签名）
  - 单元测试：domain 类型自动识别
  - 单元测试：自签名证书生成（有效证书、正确 SAN）
  - 集成测试：staging 环境 ACME 证书申请

- **提交信息**
  `feat: add ACME client and self-signed cert generation`

### 2 证书自动续期与持久化

- **功能目标**
  ACME 证书到期前自动续期，所有证书（ACME/自签名）持久化到磁盘。

- **文件结构**（创建/修改哪些文件）
  - 修改：`crates/rex-hub/src/acme.rs` — 添加续期逻辑
  - 数据目录：
    - `data/acme/` — ACME 证书和 account key
    - `data/self-signed/` — 自签名证书

- **接口设计**
  ACME 持久化：
  ```text
  data/acme/
  ├── account_key.pem      ACME account 私钥
  ├── cert.pem              当前证书
  ├── cert_key.pem          当前证书私钥
  └── cert_info.json        证书元信息（过期时间、domain、类型）
  ```

  自签名持久化：
  ```text
  data/self-signed/
  ├── ca.pem                CA 证书（用户可安装到系统信任链）
  ├── cert.pem              服务器证书
  └── cert_key.pem          服务器私钥
  ```

- **续期策略**
  | 证书类型 | 有效期 | 续期触发 | 续期方式 |
  |----------|--------|----------|----------|
  | ACME 域名 | 90 天 | 到期前 30 天 | 自动重新申请 |
  | ACME IP | 6 天 | 到期前 4 天 | 自动重新申请 |
  | 自签名 | 1 年 | 到期前 30 天 | 重新生成 |

- **交互设计**
  - 启动时检查已有证书，未过期则直接使用
  - ACME 证书即将过期 → 后台异步续期
  - 自签名证书即将过期 → 后台重新生成
  - 续期成功 → 热加载新证书（无需重启）
  - 续期失败 → 记录错误日志，下次启动重试

- **测试标准**
  - 单元测试：证书过期判断逻辑
  - 单元测试：持久化文件读写
  - 集成测试：staging 环境续期成功
  - 集成测试：证书热加载（不停机）

- **提交信息**
  `feat: add certificate renewal and persistence`

### 3 Hub 启动流程集成（优先级选择）

- **功能目标**
  Hub 启动时按优先级自动选择 TLS 模式：手动证书 > ACME > 自签名 > 无 TLS。

- **文件结构**（创建/修改哪些文件）
  - 修改：`crates/rex-hub/src/bin/rex-hub.rs` — 启动流程集成 TLS 模式选择
  - 修改：`crates/rex-hub/src/tls.rs` — 支持动态证书加载和热替换

- **接口设计**
  TLS 模式选择逻辑：
  ```text
  1. 检查 tls.cert + tls.key → 手动模式
  2. 检查 acme.domain + acme.email → ACME 模式
     a. domain 是域名 → HTTP-01 challenge
     b. domain 是 IP → TLS-ALPN-01 challenge
  3. 无 acme 配置 → 自签名模式
  4. 以上都未配置 → 无 TLS（HTTP only）
  ```

- **交互设计**
  启动日志明确显示当前模式：
  - `TLS mode: manual — using /path/to/cert.pem`
  - `TLS mode: acme — requesting cert for hub.example.com (HTTP-01)`
  - `TLS mode: acme — requesting cert for 203.0.113.1 (TLS-ALPN-01)`
  - `TLS mode: self-signed — generated cert for 192.168.1.100`
  - `TLS mode: none — HTTP only`

- **测试标准**
  - 单元测试：TLS 模式选择逻辑（4 种场景）
  - 集成测试：各模式启动成功
  - 集成测试：证书续期后热加载

- **提交信息**
  `feat: integrate TLS mode selection into Hub startup`

### 4 Docker 镜像 ACME 支持与前端状态展示

- **功能目标**
  Docker 镜像支持 ACME 配置。前端设置页展示 TLS 状态和配置指引。

- **文件结构**（创建/修改哪些文件）
  - 修改：`Dockerfile.hub` — 暴露 80 端口
  - 修改：`docker-compose.hub.yaml` — 添加 ACME 配置示例
  - 新增：后端 `GET /api/settings/tls` — TLS 状态 API
  - 修改：前端设置页 — TLS 状态面板

- **接口设计**
  docker-compose.hub.yaml：
  ```yaml
  ports:
    - "3000:3000"   # 应用端口
    - "80:80"       # ACME HTTP-01（启用 ACME 时需要）
    - "443:443"     # HTTPS（启用 TLS 时需要）
  environment:
    - REX_ACME_DOMAIN=hub.example.com
    - REX_ACME_EMAIL=admin@example.com
  ```

  前端 API：`GET /api/settings/tls`
  ```json
  {
    "mode": "acme",
    "domain": "hub.example.com",
    "status": "ready",
    "cert_expires_at": "2026-09-20T00:00:00Z",
    "cert_issuer": "Let's Encrypt",
    "last_renewal": "2026-06-22T00:00:00Z",
    "port_80_required": true
  }
  ```

- **交互设计**
  前端设置页 TLS 面板：
  - 显示当前模式（手动 / ACME 域名 / ACME IP / 自签名 / 无）
  - ACME 状态（就绪 / 续期中 / 错误）
  - 证书信息（颁发者、到期时间、上次续期）
  - 端口配置指引（如果 80/443 未暴露）

- **测试标准**
  - 文档测试：按文档步骤启用 ACME
  - 集成测试：Docker 容器 ACME 证书可用
  - 前端测试：设置页 TLS 状态正确显示

- **提交信息**
  `docs: add ACME to Docker and TLS status in settings page`
