# 0.36.1: 移动端底部导航修正

## Context

0.36.0 完成了设置页版本总览补全。PRODUCT.md 对移动端底部导航有明确的图标和布局设计要求。0.34.0 已实现底部导航栏、触摸手势、响应式布局等核心移动端功能，但底部导航图标与 PRODUCT.md 不一致，且缺少「新建」导航项。

版本类型：patch（细节修复，对齐产品设计）

## 产品边界

**做什么：**
- 底部导航图标从 emoji 改为 PRODUCT.md 定义的符号（◉ ◈ + ⬡ ⚙）
- 底部导航中间项改为「+ 新建」（打开新建连接菜单），移除工作空间导航项
- 底部导航文字标签精简（仅图标 + 简短标签）

**不做什么：**
- 新增移动端页面或后端 API
- 修改桌面端布局
- 添加新的触摸手势

## 子任务详细设计

### 1 底部导航图标和导航项修正

- **功能目标**：按 PRODUCT.md 设计修正移动端底部导航的图标和导航项
- **文件结构**：修改 `packages/rex-console-web/src/layouts/AppLayout.vue`
- **接口设计**：无 API 改动，纯前端
- **交互设计**：
  - 5 个导航项：◉ 仪表盘 | ◈ 环境 | + 新建 | ⬡ Agent | ⚙ 设置
  - 中间「+ 新建」按钮触发新建连接菜单（复用已有 Ctrl+N 逻辑）
  - 移除「工作空间」导航项（用户通过资源列表进入，不需要固定导航）
- **测试标准**：TypeScript 类型检查通过，移动端底部导航显示正确
- **提交信息**：`fix: align mobile bottom nav icons with PRODUCT.md design`
