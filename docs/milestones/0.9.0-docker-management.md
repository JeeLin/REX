# 0.9.0: Docker 容器管理

## Context

0.8.0 完成了 Redis 协议支持。当前已支持 SSH、SFTP、MySQL、PostgreSQL、Redis 五种协议。PRODUCT.md 定义了 8 种协议，其中 Docker、SQLite、S3/MinIO 尚未实现。本里程碑实现 Docker 协议支持，为用户提供容器管理能力。

版本类型：minor

## 产品边界

**做什么：**
- 新增 `rex-docker` crate，实现 Docker Engine API 通信
- Hub 端 Docker WebSocket 会话管理
- 前端 Docker 管理控制台（容器列表、日志、状态操作）
- Docker 资源创建向导（连接配置表单）
- 工作空间 Docker 面板集成

**限制：**
- Docker 连接通过 Agent 代理或直连（Unix socket / TCP）
- 仅支持单 Docker daemon 管理（不支持 Swarm 集群）
- 仅支持容器基本操作（启动/停止/重启/删除/日志/inspect）

**不做什么：**
- 不实现 Docker Compose 集成
- 不实现 Docker Swarm 集群管理
- 不实现容器内文件浏览（通过 SSH/SFTP 替代）
- 不实现镜像管理（仅容器操作）
- 不实现网络、卷、构建等功能

## 子任务详细设计

### 1 rex-docker crate（Docker Engine API + DockerConnector trait）

- **功能目标**
  创建独立的 Docker 协议 crate，实现 Docker Engine API 通信和 DockerConnector trait。遵循现有 protocol crate 的 stub 模式。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-docker/Cargo.toml`
  - 新增：`crates/rex-docker/src/lib.rs` — 导出模块
  - 新增：`crates/rex-docker/src/connector.rs` — Docker 连接器和 API 客户端
  - 修改：`Cargo.toml`（workspace）— 添加 `rex-docker` 成员和 `reqwest` 依赖
  - 修改：`crates/rex-hub/Cargo.toml` — 添加 `rex-docker` 依赖

- **接口设计**

  Docker 连接配置：
  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct DockerConfig {
      pub host: String,           // Docker daemon 地址（如 unix:///var/run/docker.sock 或 tcp://host:port）
      pub tls: Option<TlsConfig>, // TLS 配置（TCP 连接时可选）
      pub name: Option<String>,   // 实例名称
  }

  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct TlsConfig {
      pub ca_cert: Option<String>,   // CA 证书路径
      pub client_cert: String,       // 客户端证书路径
      pub client_key: String,        // 客户端私钥路径
  }
  ```

  Docker 容器信息：
  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct ContainerInfo {
      pub id: String,
      pub name: String,
      pub image: String,
      pub state: ContainerState,     // Running, Paused, Stopped, Created, Dead
      pub status: String,            // 状态描述（如 "Up 2 hours"）
      pub created: String,
      pub ports: Vec<PortMapping>,
  }

  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub enum ContainerState {
      Running,
      Paused,
      Stopped,
      Created,
      Dead,
  }

  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct PortMapping {
      pub private: u16,
      pub public: Option<u16>,
      pub protocol: String,          // tcp, udp
  }
  ```

  DockerConnector trait：
  ```rust
  #[async_trait]
  pub trait DockerConnector: Send + Sync {
      async fn connect(&mut self) -> Result<()>;
      async fn list_containers(&self, all: bool) -> Result<Vec<ContainerInfo>>;
      async fn inspect_container(&self, id: &str) -> Result<serde_json::Value>;
      async fn start_container(&self, id: &str) -> Result<()>;
      async fn stop_container(&self, id: &str) -> Result<()>;
      async fn restart_container(&self, id: &str) -> Result<()>;
      async fn remove_container(&self, id: &str) -> Result<()>;
      async fn logs(&self, id: &str, tail: u32) -> Result<String>;
      async fn info(&self) -> Result<std::collections::HashMap<String, String>>;
      async fn close(&self) -> Result<()>;
  }
  ```

  DockerConnectorImpl 方法：
  ```rust
  impl DockerConnectorImpl {
      pub fn new(config: DockerConfig) -> Self { ... }
      pub fn from_json(json: &str) -> Result<Self> { ... }
      pub fn into_config(&self) -> DockerConfig { ... }
  }
  ```

- **后端流程**
  1. `connector.rs` 实现 `DockerConnector`：
     - `connect()`：通过 Unix socket 或 TCP 连接 Docker daemon
     - `list_containers()`：调用 `GET /containers/json`，解析容器列表
     - `inspect_container()`：调用 `GET /containers/{id}/json`
     - `start_container()`：调用 `POST /containers/{id}/start`
     - `stop_container()`：调用 `POST /containers/{id}/stop`
     - `restart_container()`：调用 `POST /containers/{id}/restart`
     - `remove_container()`：调用 `DELETE /containers/{id}`
     - `logs()`：调用 `GET /containers/{id}/logs`，解析日志流
     - `info()`：调用 `GET /info`，返回 `HashMap<String, String>`（简化 JSON 为 flat map）
  2. 与 rex-redis 一致，connector 是 stub 模式，实际连接通过 Agent 代理隧道

- **依赖选型**
  - `reqwest` — HTTP 客户端（复用 workspace 依赖）
  - 无需额外 Docker SDK，直接调用 Docker Engine REST API

- **测试标准**
  - 单元测试：DockerConfig 序列化/反序列化
  - 单元测试：ContainerInfo 序列化
  - 单元测试：DockerConnector stub 行为
  - 单元测试：DockerConnector trait object safety

- **提交信息**

### 2 Hub Docker WebSocket 会话管理

- **功能目标**
  Hub 端实现 Docker 会话的 WebSocket 通道管理，支持前端通过 WebSocket 发送 Docker 命令并接收响应。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-hub/src/ws_docker.rs` — Docker WebSocket 会话管理
  - 修改：`crates/rex-hub/src/lib.rs` — 注册 `ws_docker` 模块
  - 修改：`crates/rex-hub/src/routes.rs` — 添加 Docker WebSocket 路由

- **接口设计**

  WebSocket 消息协议（Docker 频道）：
  ```text
  客户端 → Hub：
  { "type": "command", "id": "uuid", "action": "list", "params": {} }
  { "type": "command", "id": "uuid", "action": "start", "params": { "id": "container_id" } }
  { "type": "command", "id": "uuid", "action": "logs", "params": { "id": "container_id", "tail": 100 } }
  { "type": "ping" }

  Hub → 客户端：
  { "type": "response", "id": "uuid", "data": [...] }
  { "type": "error", "id": "uuid", "message": "container not found" }
  { "type": "pong" }
  { "type": "connected", "server": { "ServerVersion": "24.0.0", "OSType": "linux", "Architecture": "x86_64" } }
  { "type": "disconnected", "reason": "connection lost" }
  ```

  路由：
  ```text
  WebSocket /ws/docker/{resource_id}
  ```

- **后端流程**
  1. 前端发起 WebSocket 连接到 `/ws/docker/{resource_id}`
  2. Hub 从数据库查找 resource 配置，获取 Docker 连接信息
  3. Hub 建立到 Docker daemon 的连接（通过 Agent 代理或直连）
  4. 返回 `connected` 消息（含 Docker 系统信息）
  5. 前端发送 `command` 消息 → Hub 调用 Docker API → 返回 `response`
  6. 连接断开时返回 `disconnected` 消息

- **测试标准**
  - 单元测试：Docker WebSocket 消息序列化/反序列化
  - 单元测试：路由注册正确

- **提交信息**
  `feat: add Docker WebSocket session management in Hub`

- **交互设计**

  Docker 控制台布局：
  ```
  ┌──────────────────────────────────────────────┐
  │ ● Docker · my-server · Docker 24.0.0         │  ← 顶部状态栏
  ├──────────────────────────────────────────────┤
  │ 🔍 搜索容器...          [☐ 包含已停止] [🔄]  │  ← 搜索和过滤
  ├──────────────────────────────────────────────┤
  │ 名称            镜像              状态    操作 │  ← 表格头部
  ├──────────────────────────────────────────────┤
  │ 🟢 nginx-proxy   nginx:latest    Running  ▸   │  ← 容器行
  │ 🟢 redis-cache   redis:7-alpine  Running  ▸   │
  │ ⚫ old-app       app:v1          Exited   ▸   │  ← 已停止容器
  │ 🟡 build-agent   ubuntu:22.04    Paused   ▸   │
  ├──────────────────────────────────────────────┤
  │ 共 4 个容器 (3 运行中, 1 已停止)             │  ← 底部统计
  └──────────────────────────────────────────────┘
  ```

  交互细节：
  - 容器状态用颜色区分（🟢 Running / 🟡 Paused / ⚫ Stopped）
  - 右键菜单：启动、停止、重启、删除、查看日志、查看详情（inspect）
  - 点击容器行可展开详情面板
  - 搜索框支持容器名称、镜像名称模糊搜索
  - 「包含已停止」复选框切换显示所有容器或仅运行中
  - 日志查看器：终端风格，实时滚动，支持 tail 行数限制

  无（纯前端组件，通过 WebSocket 与 Hub 通信）

- **功能目标**
  在资源创建向导中添加 Docker 连接表单，在工作空间中将 Docker 协议映射到 DockerConsole 面板。

- **文件结构**（创建/修改哪些文件）
  - 修改：`packages/rex-console-web/src/pages/ResourceNew.vue` — 添加 Docker 连接表单步骤
  - 修改：`packages/rex-console-web/src/features/workspace/useTabs.ts` — 添加 Docker → DockerConsole 映射
  - 新增：`packages/rex-console-web/src/features/workspace/panels/WorkspaceDocker.vue` — 工作空间 Docker 面板包装

- **接口设计**

  Docker 资源创建表单字段：
  ```typescript
  interface DockerForm {
    host: string           // Docker daemon 地址
    protocol: 'unix' | 'tcp'  // 连接协议（默认 unix）
    // TCP 模式下：
    port?: number          // TCP 端口（默认 2375）
    tls?: boolean          // 是否启用 TLS
    // Unix 模式下：
    socketPath?: string    // Socket 路径（默认 /var/run/docker.sock）
    name?: string          // 实例名称
  }
  ```

  WorkspaceDocker 组件：
  ```vue
  <template>
    <DockerConsole :resource-id="resourceId" :resource-name="resourceName" />
  </template>
  ```

- **交互设计**

  资源创建向导 Docker 表单：
  - 第 2 步（连接详情）显示 Docker 特有字段
  - 连接协议切换：Unix Socket / TCP
  - Unix 模式：Socket 路径输入框（默认 /var/run/docker.sock）
  - TCP 模式：主机地址 + 端口（默认 2375）+ TLS 开关
  - 实例名称输入框（选填，placeholder: "例如：生产服务器、开发环境"）
  - 「测试连接」按钮：测试 Docker daemon 是否可达

  工作空间面板映射：
  - `useTabs.ts` 中 `protocol === 'docker'` → `WorkspaceDocker`
