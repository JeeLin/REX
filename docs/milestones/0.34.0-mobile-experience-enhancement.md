# 0.34.0: 移动端体验完善

## Context

0.33.0 完成了 SQL 编辑器体验补全。本阶段聚焦移动端体验的完善，主要是触摸操作、布局适配和性能优化。

版本类型：minor（新功能）

## 产品边界

**做什么：**
- 移动端触摸手势优化（滑动、缩放、长按）
- 响应式布局优化（不同屏幕尺寸适配）
- 移动端性能优化（减少重渲染、优化列表渲染）
- 移动端特定 UI 组件改进（浮动工具栏、汉堡菜单）
- 离线状态下的移动端体验

**不做什么：**
- 添加新的核心功能模块（如新协议支持）
- 大幅修改 desktop 布局结构
- 添加新的后端 API（除非是移动端特定必要的）

## 子任务详细设计

### 1 移动端触摸手势优化

- **功能目标**：优化移动端触摸操作体验，支持常用手势
- **文件结构**：
  - 修改：`packages/rex-console-web/src/components/WorkspaceTerminal.vue`（终端方向键、历史选择）
  - 修改：`packages/rex-console-web/src/components/ContextMenu.vue`（触摸菜单）
  - 修改：`packages/rex-console-web/src/components/ConfirmDialog.vue`（触摸确认）
  - 新增：`packages/rex-console-web/src/composables/useTouchGestures.vue`（手势处理 composable）
- **接口设计**：无 API 改动，纯前端
- **交互设计**：
  - 终端中支持上下滑动切换历史记录
  - 左右滑动在终端中切换光标位置（或根据上下文实现其他功能）
  - 长按触发上下文菜单
  - 双击放大/缩小终端字体
  - 触摸滑动切换工作空间面板
- **后端流程**：无
- **测试标准**：
  - 在移动设备模拟器上测试手势识别准确率
  - 触摸操作延迟 < 100ms
  - 手势冲突正确处理（如滚动vs滑动）
- **提交信息**：`feat: add mobile touch gestures support`

### 2 响应式布局优化

- **功能目标**：优化不同移动设备尺寸下的布局适配
- **文件结构**：
  - 修改：`packages/rex-console-web/src/layouts/AppLayout.vue`（布局适配逻辑）
  - 修改：`packages/rex-console-web/src/styles/base.css`（响应式断点和样式）
  - 修改：`packages/rex-console-web/src/components/EnvironmentEditModal.vue`（移动端表单适配）
  - 修改：`packages/rex-console-web/src/pages/ResourceNew.vue`（移动端向导适配）
- **接口设计**：无 API 改动，纯前端
- **交互设计**：
  - 小屏幕（< 480px）：单栏布局，底部导航栏
  - 中屏幕（480px-768px）：两栏布局，可折叠侧边栏
  - 大屏幕（> 768px）：保持现有 desktop 布局
  - 表单元素在移动端自动调整大小和间距
  - 表格在移动端自动切换为卡片布局（已有部分实现需完善）
- **后端流程**：无
- **测试标准**：
  - 在不同断点下布局正确切换
  - 所有交互元素在移动端可达且易用
  - 文字可读性符合 WCAG AA 标准
- **提交信息**：`feat: enhance mobile responsive layout`

### 3 移动端性能优化

- **功能目标**：减少移动端重渲染，优化长列表性能
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/workersql/SqlResults.vue`（虚拟滚动或分页优化）
  - 修改：`packages/rex-console-web/src/features/sql/SqlSidebar.vue`（树形虚拟滚动）
  - 修改：`packages/rex-console-web/src/features/workspace/panels/WorkspaceSql.vue`（查询历史虚拟列表）
  - 新增：`packages/rex-console-web/src/composables/useVirtualList.vue`（虚拟列表 composable）
- **接口设计**：无 API 改动，纯前端
- **交互设计**：
  - 长列表（如查询历史、文件列表）使用虚拟滚动只渲染可见项
  - 防抖处理频繁触发的事件（如搜索输入、窗口 resize）
  - 图片和图标适当延迟加载
  - 移动端降低动画频率和复杂度
- **后端流程**：无
- **测试标准**：
  - 1000+ 项列表在移动设备上滚动流畅（帧率 > 30fps）
  - 内存占用不随列表长度线性增长
  - 长时间使用无明显卡顿
- **提交信息**：`feat: optimize mobile performance with virtual lists`

### 4 移动端特定 UI 组件改进

- **功能目标**：改进移动端特有的 UI 组件和交互模式
- **文件结构**：
  - 修改：`packages/rex-console-web/src/components/ToastProvider.vue`（移动端 toast 位置和样式）
  - 修改：`packages/rex-console-web/src/components/LoadingSpinner.vue`（移动端适配尺寸）
  - 修改：`packages/rex-console-web/src/components/EnvironmentEditModal.vue`（移动端表单验证和提交）
  - 修改：`packages/rex-console-web/src/layouts/AppLayout.vue`（汉堡菜单和底部导航）
- **接口设计**：无 API 改动，纯前端
- **交互设计**：
  - Toast 在移动端底部显示，避免遮挡重要内容
  - 加载指示器尺寸适合手指操作
  - 表单验证实时反馈，错误信息易读
  - 底部导航栏（Home, Dashboard, Environments, Agents, Settings）快速切换
  - 汉堡菜单侧滑打开侧边栏
  - 移动端专用的空状态页面和引导页
- **后端流程**：无
- **测试标准**：
  - 所有移动端特定组件在 iOS 和 Android 主流浏览器中正常工作
  - 触摸目标尺寸符合指南（最小 48x48dp）
  - 无横向滚动条（除非特殊需要如图片画廊）
- **提交信息**：`feat: improve mobile-specific UI components`
