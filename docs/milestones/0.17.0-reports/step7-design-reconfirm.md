## 实际 API 设计对比

里程碑文档中的 API 设计：
```
Body: { "databases": [{ "id", "name", "type", "host", "port", "username", "password", "database" }] }
```

实际实现（安全改进）：
```
Body: { "resource_ids": ["res_001", "res_002"] }
```

**说明**：代码审查中发现前端传密码不安全，改为 resource_id 方式，后端从数据库加载配置。这是正确的安全改进，符合产品文档中「自托管」的数据安全原则。
