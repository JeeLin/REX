# 0.29.0: 环境管理与前端交互补全

## Context

前序版本（0.28.0）移除了 Docker 协议，将协议从 8 个精简为 7 个。当前所有 7 种协议的连接器已实现（Redis 为 stub，其余为真实连接），前端各页面均有基本实现。

但环境管理和多个页面的前端交互存在"半成品"状态：环境无法编辑/删除、资源状态硬编码、大量右键菜单项只有标签没有实际操作。本版本聚焦将这些交互打磨到完整可用。

版本类型：minor（新功能：环境编辑/删除、资源探活、交互补全）

## 产品边界

**做什么：**
- 环境编辑与删除（API + 前端弹窗）
- 资源编辑与删除（API + 前端弹窗）
- 资源轻量探活（ping 端点 + 前端状态显示）
- 环境卡片信息补全（资源数、Agent数、类型分布、最后活跃时间）
- 右键菜单 action 补全（Dashboard/Environments/EnvironmentDetail/AuditLog）
- i18n key 同步 + 硬编码字符串替换
- 前端 bug 修复（SSH 表单、步骤指示器、资源状态）

**不做什么：**
- Redis 真实连接实现（留给后续版本）
- 新协议支持
- 前端架构重构

## 子任务详细设计

### 1 环境编辑与删除

- **功能目标**：用户可在环境详情页或环境卡片上编辑名称/描述、删除环境（需确认弹窗）
- **文件结构**：
  - 后端：`crates/rex-hub/src/routes.rs`（✅ 已有环境 PUT/DELETE API，无需修改）
  - 前端：`packages/rex-console-web/src/components/EnvironmentEditModal.vue`（新建）
  - 前端：`packages/rex-console-web/src/pages/EnvironmentDetail.vue`（修改）
  - 前端：`packages/rex-console-web/src/pages/Environments.vue`（修改）
  - 前端：`packages/rex-console-web/src/pages/Dashboard.vue`（修改）
- **接口设计**（已存在，仅需前端对接）：
  - `PUT /api/environments/:id` — 更新环境名称、描述
  - `DELETE /api/environments/:id` — 删除环境（级联删除关联资源）
- **交互设计**：
  - 编辑弹窗：模态面板，预填当前名称和描述，底部「取消」「保存」按钮
  - 删除确认：模态弹窗，提示「删除环境将同时删除该环境下的所有资源，此操作不可撤销」，红色确认按钮
  - 参考原型 `prototype/environment.html` 第 193-194 行的编辑/删除按钮位置
- **后端流程**：
  - 更新：验证名称非空，执行 UPDATE environments，写入审计日志
  - 删除：查询关联资源数，级联 DELETE resources WHERE environment_id = ?，再 DELETE environments，写入审计日志
- **测试标准**：
  - 更新名称/描述成功，返回更新后的数据
  - 删除空环境成功
  - 删除有资源的环境 → 资源被级联删除
  - 删除不存在的环境 → 404
- **提交信息**：`feat: add environment edit and delete API + frontend`

### 2 资源编辑与删除

- **功能目标**：用户可在环境详情页编辑资源名称和配置、删除资源
- **文件结构**：
  - 后端：`crates/rex-hub/src/routes.rs`（✅ 已有资源 PUT/DELETE API，无需修改）
  - 前端：`packages/rex-console-web/src/components/ResourceEditModal.vue`（✅ 已存在，需修改完善：补充协议特定字段、修正交互）
  - 前端：`packages/rex-console-web/src/pages/EnvironmentDetail.vue`（修改）
- **接口设计**（已存在，仅需前端对接）：
  - `PUT /api/environments/:env_id/resources/:id` — 更新资源配置（名称、连接参数）
  - `DELETE /api/environments/:env_id/resources/:id` — 删除资源
- **交互设计**：
  - 编辑弹窗：修改已有的 ResourceEditModal.vue，确保预填当前值，底部「取消」「保存」按钮
  - 删除确认：「删除资源后，相关的查询文件和历史记录将保留，但连接将断开」
- **后端流程**（API 已实现，无需修改）：
  - 更新：验证配置合法性，加密凭据字段，执行 UPDATE resources，审计日志
  - 删除：DELETE resources WHERE id = ?，审计日志
- **测试标准**：
  - 更新资源名称成功
  - 更新密码/凭据成功（重新加密存储）
  - 删除资源成功
- **提交信息**：`feat: enhance resource edit modal and wire up delete`

### 3 资源状态实时检查

- **功能目标**：环境详情页的资源列表显示真实连接状态，而非硬编码 "Online"
- **文件结构**：
  - 后端：`crates/rex-hub/src/routes.rs`（新增轻量探活接口）
  - 前端：`packages/rex-console-web/src/pages/EnvironmentDetail.vue`（修改）
- **接口设计**：
  - `POST /api/environments/:env_id/resources/:id/ping` — 轻量连接测试
  - 响应：`{ "status": "online" | "offline", "latency_ms": 12 }`
- **后端流程**：
  - 根据协议类型尝试 TCP 连接到目标主机:端口
  - SSH/MySQL/PostgreSQL：TCP 握手 + 协议 banner 读取（超时 3 秒）
  - Redis/SQLite/S3：TCP 连接测试（超时 3 秒）
  - 返回连接状态和延迟
- **测试标准**：
  - 对在线服务器 ping → online + 合理延迟
  - 对不存在的服务器 ping → offline
  - 超时处理（3 秒内返回）
- **提交信息**：`feat: add resource ping endpoint for real status check`

### 4 环境卡片信息补全

- **功能目标**：环境列表和详情页的卡片显示完整信息
- **文件结构**：
  - 后端：`crates/rex-hub/src/routes.rs`（环境列表 API 响应增强）
  - 前端：`packages/rex-console-web/src/pages/Environments.vue`（修改）
  - 前端：`packages/rex-console-web/src/pages/EnvironmentDetail.vue`（修改）
  - 前端：`packages/rex-console-web/src/pages/Dashboard.vue`（修改）
- **接口设计**：
  - `GET /api/environments` 响应增加：`resource_count`、`agent_count`、`resource_types`（协议分布 map）、`connection_modes`（{ direct: N, agent: N }）、`last_active_at`
- **交互设计**：
  - 参考原型 `prototype/environments.html` 的环境卡片布局
  - 卡片底部显示：`SSH ×2 · MySQL ×1` 资源类型分布
  - 右下角显示：`◉ 直连: 2 · ⬡ Agent: 1` 连接方式统计
- **后端流程**：
  - 环境列表查询 JOIN 资源表和 Agent 表，聚合统计
  - `last_active_at` 从审计日志取该环境最近操作时间
- **测试标准**：
  - 有 2 个 SSH + 1 个 MySQL 的环境 → `resource_types: { ssh: 2, mysql: 1 }`
  - 有 Agent 在线 → `agent_count: 1`
  - 有最近操作 → `last_active_at` 非空
- **提交信息**：`feat: enrich environment list API with resource stats`
