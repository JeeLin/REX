# 0.22.0: Agent 管理完善

## Context

0.21.0 完成了 Agent 自动更新流程。此前 0.12.0 已实现 Agent 管理基础（列表、配置弹窗、部署指南、令牌重置）。但 Agent 日志查看器仍使用 mock 数据，且缺少远程重启能力。本里程碑对接真实日志 API 和重启指令，完善 Agent 管理体验。

版本类型：minor（新功能，向后兼容），版本号 0.22.0。

## 产品边界

**做什么：**
- Agent 通过心跳上报近期日志，Hub 存储并提供查询 API
- 前端 Agent 日志查看器对接真实日志数据
- Hub 提供 Agent 远程重启 API，前端增加重启按钮
- Agent 响应重启指令，通过 supervisor 完成重启

**不做什么：**
- 不实现日志持久化到文件（内存存储即可，覆盖最近 1 小时）
- 不实现日志搜索/导出
- 不修改 WebSocket 协议基础结构（在现有消息类型上扩展）
- 不修改自动更新流程（已完成）

## 子任务详细设计

### 1 Agent 日志上报 + Hub 存储与查询 API

**功能目标：**
Agent 收集运行日志，通过心跳上报给 Hub。Hub 存储最近 1 小时的日志，提供按 Agent 查询的 API。

**文件结构：**
```text
crates/
├── rex-agent/
│   └── src/
│       ├── log_collector.rs   新增：环形缓冲区日志收集器
│       └── ws.rs              修改：心跳 payload 新增 recent_logs 字段
├── rex-hub/
│   └── src/
│       ├── agent.rs           修改：新增 get_agent_logs handler
│       └── routes.rs          修改：新增日志查询路由
```

**接口设计：**

Agent 心跳 payload 新增 `recent_logs` 字段（增量，仅包含上次心跳后新增的日志）：
```json
{
  "msg_type": "heartbeat",
  "payload": {
    "agent_id": "...",
    "version": "...",
    "auto_update": true,
    "recent_logs": [
      {
        "timestamp": "2026-06-26T16:51:30Z",
        "level": "info",
        "message": "TLS 握手完成"
      }
    ]
  }
}
```

`GET /api/agents/:agent_id/logs` → 返回指定 Agent 的最近日志
```json
{
  "data": {
    "logs": [
      {
        "timestamp": "2026-06-26T16:51:30Z",
        "level": "info",
        "message": "TLS 握手完成"
      }
    ],
    "total": 42
  }
}
```

查询参数：`?since=<ISO timestamp>`（可选，返回指定时间之后的日志）

**后端流程：**
1. `log_collector.rs`：实现 `LogCollector`，基于 `VecDeque` 的环形缓冲区，最多存储 1000 条日志，自动淘汰超过 1 小时的条目
2. `ws.rs`：Agent 启动时注册自定义 `tracing` Layer 到 `LogCollector`，每次心跳时 drain 新增日志附到 payload
3. Hub `ws.rs`：解析心跳中的 `recent_logs`，存入内存中的 `AgentLogStore`（`HashMap<agent_id, VecDeque<LogEntry>>`，每 agent 最多 1000 条）
4. `agent.rs`：`get_agent_logs` handler，从 `AgentLogStore` 读取，支持 `since` 参数过滤
5. `routes.rs`：注册 `GET /api/agents/:agent_id/logs`

**设计原则**：日志仅存内存，Hub 重启后日志清空。足够覆盖自托管场景的"查看最近运行状态"需求。

```

```

### 2 Agent 远程重启 API

**功能目标：**
Hub 通过 WebSocket 向 Agent 发送重启指令，Agent 收到后优雅退出并由 supervisor 重启。

**文件结构：**
```text
crates/
├── rex-hub/
│   └── src/
│       ├── agent.rs           修改：新增 restart_agent handler
│       └── routes.rs          修改：新增重启路由
├── rex-agent/
│   └── src/
│       └── ws.rs              修改：处理 restart 消息类型
```

**接口设计：**

`POST /api/agents/:agent_id/restart` → 向 Agent 发送重启指令
```json
// Response
{
  "data": {
    "message": "restart command sent"
  }
}
```

WebSocket 新增消息类型：
```json
{
  "msg_type": "restart",
  "payload": {}
}
```

**后端流程：**
1. Hub `agent.rs`：`restart_agent` handler 找到 Agent 的 WebSocket 连接，发送 `restart` 消息
2. Hub `ws.rs`：维护 `agent_connections: HashMap<agent_id, Sender<Message>>`，在 Agent 断开时清理
3. Agent `ws.rs`：匹配 `restart` 消息类型，调用 `std::process::exit(10)`（复用 supervisor 更新流程中的重启语义）
4. `routes.rs`：注册 `POST /api/agents/:agent_id/restart`

**提交信息：**

### 3 前端对接（日志查看器真实数据 + 重启按钮）

**功能目标：**
- Agent 日志查看器替换 mock 数据为真实 API 轮询
- Agent 管理页面增加重启按钮（带确认）

**文件结构：**
```text
packages/rex-console-web/src/
├── features/agents/
│   ├── AgentLogModal.vue     修改：对接真实日志 API
│   ├── AgentCard.vue         修改：增加重启按钮
│   └── AgentStatusPanel.vue  修改：增加重启操作
└── api/
    └── agent.ts              修改：新增 getAgentLogs、restartAgent API
```

**交互设计：**
- AgentLogModal：打开时调用 `GET /api/agents/:id/logs`，每 5 秒轮询增量日志（`?since=<last_timestamp>`）
- Agent 卡片操作按钮：增加"重启"按钮，点击后弹出确认弹窗
- 确认弹窗：提示"重启将短暂中断该 Agent 代理的所有连接"，确认后调用 `POST /api/agents/:id/restart`

## 设计核对点

- [ ] 日志上报不增加心跳 payload 大小负担（仅增量日志）
- [ ] 日志存储不引入新数据库表（内存存储）
- [ ] 重启指令通过现有 WebSocket 通道发送
- [ ] Agent 重启复用 supervisor 模型（exit code 10）
- [ ] 前端日志轮询间隔合理（5 秒）
- [ ] 重启确认弹窗防止误操作
- [ ] 没有引入多用户、RBAC 概念
