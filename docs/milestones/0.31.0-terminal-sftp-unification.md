# 0.31.0: 终端 SFTP 体验统一与功能补全

## Context

0.30.0 完成了 SSH 密钥上传、编码/保活选项、设置行为对接和仪表盘统计。当前终端内置 SFTP 面板存在两套实现：`TerminalSftp.vue`（独立终端页使用）功能较全但缺 rename 和"发送到"，`WorkspaceTerminal.vue` 内嵌简化版 SFTP 只有基础文件列表。需要统一为一个组件并补全缺失功能。

版本类型：minor（功能补全：rename、跨连接传输、组件统一）

## 产品边界

**做什么：**
- TerminalSftp.vue 补全 rename（重命名）功能
- TerminalSftp.vue 补全"发送到"跨连接传输功能
- WorkspaceTerminal.vue 内嵌 SFTP 改为复用 TerminalSftp.vue 组件
- 移动端 SFTP 适配

**不做什么：**
- 新协议支持
- 多用户/RBAC
- 文件内容预览/编辑

## 子任务详细设计

### 1 TerminalSftp 补全 rename 功能

- **功能目标**：SFTP 面板支持文件/文件夹重命名，与 Files.vue 保持一致的交互方式
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/terminal/TerminalSftp.vue`
- **接口设计**：
  - 复用 `api/files.ts` 中已有的 `renameFile(resourceId, oldPath, newPath)` 函数
- **交互设计**（参考 PRODUCT.md §3.8）：
  - 右键菜单新增「重命名」项（与「复制路径」「删除」同级）
  - 点击后当前文件名变为可编辑 input，选中文件名部分（不含扩展名）
  - Enter 确认、Esc 取消
  - 重命名成功后刷新文件列表
  - 移动端：长按或右键菜单触发
- **后端流程**：无需后端改动，复用已有 API
- **测试标准**：
  - 右键文件 → 重命名 → 输入新名称 → Enter → 文件名更新
  - 右键文件夹 → 重命名 → 同上
  - Esc 取消 → 名称不变
  - 重命名到已存在名称 → 显示错误提示
- **提交信息**：`feat: add rename to terminal SFTP panel`

### 2 TerminalSftp 补全"发送到"跨连接传输

- **功能目标**：终端 SFTP 面板支持"发送到"功能，将文件发送到其他已打开的连接
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/terminal/TerminalSftp.vue`
- **接口设计**：
  - 复用 `api/transfer.ts` 中已有的 `createTransfer(source, target)` 函数
  - 复用 `features/workspace/useTabs.ts` 获取当前已打开的标签列表（过滤出支持文件传输的标签）
- **交互设计**（参考 PRODUCT.md §3.8 跨连接文件传输）：
  - 右键菜单新增「发送到…」项（仅文件，不含目录）
  - 点击后弹出目标选择弹窗：列出所有已打开的支持文件传输的标签（SFTP、SSH、S3）
  - 选择目标 → 确认 → 调用 createTransfer API → Toast 提示传输已启动
  - 传输进度通过已有的 `useTransferQueue` composable 展示
- **后端流程**：无需后端改动，复用已有 API
- **测试标准**：
  - 右键文件 → 发送到 → 弹出目标列表
  - 选择目标 → 确认 → Toast 提示成功
  - 无其他文件传输标签时 → 菜单项不显示
- **提交信息**：`feat: add cross-connection transfer to terminal SFTP panel`

### 3 WorkspaceTerminal 复用 TerminalSftp 组件

- **功能目标**：消除 `WorkspaceTerminal.vue` 中独立实现的简化版 SFTP，改为使用 `TerminalSftp.vue`，统一功能和视觉
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/workspace/panels/WorkspaceTerminal.vue`
  - 删除/简化内联 SFTP 代码（状态变量、loadSftpFiles、sftpGoUp、sftpOpenDir、分隔条逻辑、SFTP 相关 CSS）
- **交互设计**：
  - 移除 `WorkspaceTerminal.vue` 中内联的 `FileList` SFTP 面板
  - 改为 `import TerminalSftp from '@/features/terminal/TerminalSftp.vue'`，使用与 `Terminal.vue` 相同的 split view 模式
  - 工具栏 SFTP 按钮（Ctrl+Shift+F）行为不变
  - 保持分隔条拖拽调整宽度（复用已有的 startResize/onResize 逻辑）
  - 保持拖拽文件到终端区域（handleDrop/handleDragOver）
  - 保持移动端浮动工具栏不变
- **后端流程**：无需后端改动
- **测试标准**：
  - 点击 SFTP 按钮 → 右侧弹出 TerminalSftp 面板（非简化版）
  - 拖拽分隔条 → 调整宽度
  - 重命名、上传、下载、发送到等功能均可用
  - type-check 通过
- **提交信息**：`refactor: unify workspace terminal SFTP with TerminalSftp component`
