# 0.32.0: 传输进度体验补全

## Context

0.31.0 统一了终端 SFTP 体验。现有传输系统（API、TransferManager、前端 TransferQueuePanel/TransferItem）已可用，但存在以下体验缺陷：TransferItem 状态文本硬编码中文无 i18n、传输完成无 Toast 通知、终端 SFTP 面板缺少传输队列入口、无传输速度/剩余时间显示。

版本类型：minor（功能补全）

## 产品边界

**做什么：**
- TransferItem 状态文本 i18n 化
- 传输完成/失败 Toast 通知
- TerminalSftp 面板显示传输队列
- TransferItem 补充传输速度和预计剩余时间显示

**不做什么：**
- 新协议支持
- 冲突处理弹窗（产品文档提及但后端未实现，不在本阶段范围）
- 传输队列独立页面

## 子任务详细设计

### 1 TransferItem i18n 化 + 传输速度/剩余时间

- **功能目标**：TransferItem 所有状态文本使用 i18n key，并在运行中任务显示传输速度和预计剩余时间
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/files/TransferItem.vue`
  - 修改：`packages/rex-console-web/src/i18n/zh.ts`、`packages/rex-console-web/src/i18n/en.ts`
- **接口设计**：
  - 复用现有 `TransferTask` 类型（含 `progress: { total_bytes, transferred_bytes }`）
  - 前端通过追踪 transferred_bytes 快照计算速度（bytes/s），无需后端改动
- **交互设计**：
  - 状态文本 i18n：「完成」→ `t('files.transfer.completed')`、「等待中」→ `t('files.transfer.pending')` 等
  - 源/目标标签 i18n：「源：」→ `t('files.transfer.source')`、「目标：」→ `t('files.transfer.target')`、「错误：」→ `t('files.transfer.error')`
  - 传输速度显示：在进度条旁显示 `1.2 MB/s`
  - 剩余时间显示：在速度旁显示 `剩余 2:30` 或 `剩余 < 1 分钟`
  - 计算方式：在 `useTransferQueue` 的3秒轮询回调中记录 transferred_bytes 快照，相邻两次轮询差值除以间隔得到速度（bytes/s）
- **后端流程**：无需后端改动
- **测试标准**：
  - TransferItem 在 pending/running/completed/failed/cancelled 状态显示正确 i18n 文本
  - 运行中任务显示传输速度（格式化为 KB/s 或 MB/s）
  - 运行中任务显示预计剩余时间（格式化为 m:ss 或 < 1 分钟）
  - 切换语言后文本正确切换
- **提交信息**：`feat: add i18n and speed/ETA to transfer item`

### 2 传输完成 Toast 通知 + TerminalSftp 传输队列入口

- **功能目标**：传输完成或失败时弹出 Toast 通知；TerminalSftp 面板底部显示传输队列，与 Files.vue 保持一致
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/files/useTransferQueue.ts`（导出 prevTasks 用于状态变化检测）
  - 修改：`packages/rex-console-web/src/pages/Files.vue`（watch transferTasks 变化触发 Toast）
  - 修改：`packages/rex-console-web/src/features/terminal/TerminalSftp.vue`（添加 TransferQueuePanel）
  - 修改：`packages/rex-console-web/src/i18n/zh.ts`、`packages/rex-console-web/src/i18n/en.ts`
- **接口设计**：
  - `useTransferQueue` 导出 `prevTasks: Ref<TransferTask[]>`（上一次轮询快照），消费方通过 watch 对比检测状态变化
  - Files.vue 和 TerminalSftp.vue 各自 watch `transferTasks`，对比 `prevTasks` 检测新完成/失败的任务并触发 Toast
- **交互设计**：
  - TerminalSftp 底部显示折叠式传输队列（与 Files.vue 相同的 TransferQueuePanel）
  - 传输完成 → Toast `t('files.transfer.completedToast', { file: filename })`
  - 传输失败 → Toast `t('files.transfer.failedToast', { file: filename, error: detail })`
- **后端流程**：无需后端改动
- **测试标准**：
  - 传输完成时显示成功 Toast
  - 传输失败时显示失败 Toast
  - TerminalSftp 面板底部显示传输队列，可折叠/展开
  - 传输队列中可取消和移除任务
- **提交信息**：`feat: add transfer toast notifications and queue panel to terminal SFTP`
