# 0.10.0: SQLite 协议支持

## Context

0.9.0 完成了 Docker 协议支持。当前已支持 SSH、SFTP、MySQL、PostgreSQL、Redis、Docker 六种协议。PRODUCT.md 定义了 8 种协议，其中 SQLite 和 S3/MinIO 尚未实现。本里程碑实现 SQLite 协议支持，让用户可以通过 REX Hub 管理本地 SQLite 数据库。

版本类型：minor

## 产品边界

**做什么：**
- 新增 `rex-sqlite` crate，实现 SQLite 数据库连接和操作
- Hub 端 SQLite WebSocket 会话管理
- 前端复用现有 SQL 控制台（`features/sql/`），适配 SQLite 特有行为
- SQLite 资源创建向导（数据库文件路径表单）
- 工作空间 SQLite 面板集成

**限制：**
- SQLite 数据库文件位于 Hub 服务器本地或 Agent 机器上
- 仅支持只读查询和基本写入操作（不支持数据库创建/删除）
- 单文件数据库，不支持 WAL 模式的高级配置

**不做什么：**
- 不实现数据库文件创建/删除管理
- 不实现数据库迁移工具
- 不实现多数据库同时连接
- 不实现 SQLite 特有的高级功能（FTS、JSON 函数等的特殊 UI）

### 1 rex-sqlite crate（SQLiteConnector trait + 文件连接）

- **功能目标**
  创建独立的 SQLite 协议 crate，实现 SQLite 数据库连接和查询操作。遵循现有 protocol crate 的 stub 模式。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-sqlite/Cargo.toml`
  - 新增：`crates/rex-sqlite/src/lib.rs` — 导出模块
  - 新增：`crates/rex-sqlite/src/connector.rs` — SQLite 连接器
  - 修改：`Cargo.toml`（workspace）— 添加 `rex-sqlite` 成员
  - 修改：`crates/rex-hub/Cargo.toml` — 添加 `rex-sqlite` 依赖

- **接口设计**

  SQLite 连接配置：
  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct SqliteConfig {
      pub db_path: String,   // SQLite 数据库文件路径
      pub name: Option<String>,  // 实例名称
  }
  ```

  SQLiteConnector trait：
  ```rust
  #[async_trait]
  pub trait SqliteConnector: Send + Sync {
      async fn connect(&mut self) -> Result<()>;
      async fn execute(&self, sql: &str) -> Result<SqliteResult>;
      async fn list_tables(&self) -> Result<Vec<String>>;
      async fn get_table_info(&self, table: &str) -> Result<Vec<ColumnInfo>>;
      async fn close(&self) -> Result<()>;
  }
  ```

  查询结果：
  ```rust
  #[derive(Debug, Serialize, Deserialize)]
  pub struct SqliteResult {
      pub columns: Vec<String>,
      pub rows: Vec<Vec<serde_json::Value>>,
      pub affected_rows: u64,
      pub elapsed_ms: u64,
  }
  ```

  SqliteConnectorImpl 方法：
  ```rust
  impl SqliteConnectorImpl {
      pub fn new(config: SqliteConfig) -> Self { ... }
      pub fn from_json(json: &str) -> Result<Self> { ... }
      pub fn into_config(&self) -> SqliteConfig { ... }
  }
  ```

- **后端流程**
  1. `connector.rs` 实现 `SqliteConnector`：
     - `connect()`：验证数据库文件存在且可读
     - `execute()`：执行 SQL 查询，返回结构化结果
     - `list_tables()`：查询 `sqlite_master` 获取所有表名
     - `get_table_info()`：查询 `PRAGMA table_info` 获取列信息
     - `close()`：关闭连接
  2. 与 rex-redis/rex-docker 一致，connector 是 stub 模式，实际连接通过 Agent 代理隧道

- **依赖选型**
  - `rusqlite` — SQLite 客户端（workspace 已有依赖）
  - 无需额外 crate

- **测试标准**
  - 单元测试：SqliteConfig 序列化/反序列化
  - 单元测试：SqliteConnector stub 行为
  - 单元测试：SqliteConnector trait object safety
  - 单元测试：SqliteResult 序列化

- **提交信息**
  `feat: add rex-sqlite crate with SqliteConnector trait`

### 2 Hub SQLite WebSocket 会话管理

- **功能目标**
  Hub 端实现 SQLite 会话的 WebSocket 通道管理，支持前端通过 WebSocket 发送 SQL 查询并接收响应。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-hub/src/ws_sqlite.rs` — SQLite WebSocket 会话管理
  - 修改：`crates/rex-hub/src/lib.rs` — 注册 `ws_sqlite` 模块
  - 修改：`crates/rex-hub/src/routes.rs` — 添加 SQLite WebSocket 路由

- **接口设计**

  WebSocket 消息协议（SQLite 频道）：
  ```text
  客户端 → Hub：
  { "type": "command", "id": "uuid", "action": "execute", "params": { "sql": "SELECT * FROM users" } }
  { "type": "command", "id": "uuid", "action": "tables", "params": {} }
  { "type": "command", "id": "uuid", "action": "columns", "params": { "table": "users" } }
  { "type": "ping" }

  Hub → 客户端：
  { "type": "response", "id": "uuid", "data": { "columns": [...], "rows": [...], "affected_rows": 0, "elapsed_ms": 1 } }
  { "type": "error", "id": "uuid", "message": "table not found" }
  { "type": "pong" }
  { "type": "connected", "server": { "db_path": "/path/to/db.sqlite", "size": "12345" } }
  { "type": "disconnected", "reason": "connection lost" }
  ```

  路由：
  ```text
  WebSocket /ws/sqlite/:resource_id
  ```

- **后端流程**
  1. 前端发起 WebSocket 连接到 `/ws/sqlite/:resource_id`
  2. Hub 从数据库查找 resource 配置，获取 SQLite 文件路径
  3. Hub 建立到 SQLite 数据库的连接（通过 Agent 代理或直连）
  4. 返回 `connected` 消息（含数据库路径和文件大小）
  5. 前端发送 `command` 消息 → Hub 执行 SQL → 返回 `response`
  6. 连接断开时返回 `disconnected` 消息

- **测试标准**
  - 单元测试：SQLite WebSocket 消息序列化/反序列化
  - 单元测试：路由注册正确

- **提交信息**
  `feat: add SQLite WebSocket session management in Hub`
