# 0.23.0: 修复非 SSH 资源连接

## Context

SSH 终端（Terminal.vue / WorkspaceTerminal.vue）连接正常，但 Redis、Docker、S3、SQLite 四种资源类型在工作空间打开后连接失败（WebSocket URL 中 token 参数为空）。根因是 token localStorage key 不一致：登录存 `rex-token`，但 4 个 session composable 读 `token`。此外 Docker/S3/SQLite 缺少 `onMounted` 自动连接，需要手动点击"连接"按钮。

版本类型：patch（bug 修复），版本号 0.23.0。

## 产品边界

**做什么：**
- 修复 Redis/Docker/S3/SQLite WebSocket 连接的 token key 不一致
- Docker/S3/SQLite 在工作空间打开时自动连接（对齐 SSH 行为）
- Redis 已有 `onMounted` 自动连接，修复 token 后即可工作

**不做什么：**
- 不修改 MySQL/PostgreSQL 连接（走 HTTP REST API，不受影响）
- 不修改后端代码（后端 WebSocket handler 正确验证 token）
- 不新增协议支持
- 不修改工作空间标签打开逻辑

## 子任务详细设计

### 1 修复 4 个 session composable 的 token key

**功能目标：**
修复 `useRedisSession.ts`、`useDockerSession.ts`、`useS3Session.ts`、`useSqliteSession.ts` 中 WebSocket URL 的 token 读取，从 `localStorage.getItem('token')` 改为 `localStorage.getItem('rex-token')`，与 auth store 保持一致。

**文件结构：**
```text
packages/rex-console-web/src/
├── features/redis/
│   └── useRedisSession.ts      修改：token key → rex-token
├── features/docker/
│   └── useDockerSession.ts     修改：token key → rex-token
├── features/s3/
│   └── useS3Session.ts         修改：token key → rex-token
└── features/sqlite/
    └── useSqliteSession.ts     修改：token key → rex-token
```

**接口设计：**
每个文件只改一行：`localStorage.getItem('token')` → `localStorage.getItem('rex-token')`

**参考：**
- `Terminal.vue:188`：`const token = localStorage.getItem('rex-token') ?? ''` ✅
- `WorkspaceTerminal.vue:250`：`const token = localStorage.getItem('rex-token') ?? ''` ✅
- `api/client.ts:12`：`const token = localStorage.getItem('rex-token')` ✅

**提交信息：**
```
fix: use correct token key in redis/docker/s3/sqlite session composables
```

### 2 Docker/S3/SQLite 添加 onMounted 自动连接

**功能目标：**
DockerConsole.vue、S3Console.vue、WorkspaceSqlite.vue 在组件挂载时自动调用 `connect()`，对齐 SSH 和 Redis 的行为。用户打开工作空间标签后无需手动点击"连接"按钮。

**文件结构：**
```text
packages/rex-console-web/src/
├── features/docker/
│   └── DockerConsole.vue       修改：onMounted 中添加 auto-connect
├── features/s3/
│   └── S3Console.vue           修改：onMounted 中添加 auto-connect
└── workspace/panels/
    └── WorkspaceSqlite.vue     修改：onMounted 中添加 auto-connect
```

**交互设计：**
参考 RedisConsole.vue:192-200 的模式：
```typescript
onMounted(() => {
  if (!session.connected.value) {
    session.connect().catch(() => {})
  }
})
```

Docker 和 S3 在连接后需要加载初始数据（容器列表 / bucket 列表），在 connect 回调中追加：
- DockerConsole：`session.connect().then(() => refreshContainers()).catch(() => {})`
- S3Console：`session.connect().then(() => refreshBuckets()).catch(() => {})`
- WorkspaceSqlite：`session.connect().then(() => refreshTables()).catch(() => {})`

**提交信息：**
```
fix: auto-connect docker/s3/sqlite on workspace mount
```
