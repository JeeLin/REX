# 0.5.0: Docker Compose 部署配置

## Context

v0.4.0 完成了设置项连接实际行为。当前项目有 `Dockerfile.hub` 和 `Dockerfile.agent`，但缺少 docker-compose 配置文件。PRODUCT.md 中的部署说明只有 `docker run` 命令，用户需要 docker-compose.yaml 来简化部署流程。

本阶段添加 Hub 和 Agent 的 docker-compose 配置，并更新部署文档。

## 子任务详细设计

### 1 Hub docker-compose.yaml

**功能目标：**
创建 `docker-compose.hub.yaml`，提供 Hub 的标准 docker-compose 部署配置。

**文件结构：**
```text
docker-compose.hub.yaml     新建
```

**配置设计：**
```yaml
services:
  hub:
    build:
      context: .
      dockerfile: Dockerfile.hub
    image: rexhub/console:latest
    container_name: rex-hub
    restart: unless-stopped
    ports:
      - "3000:3000"
    volumes:
      - hub-data:/app/data
    environment:
      - REX_SECRET_KEY=${REX_SECRET_KEY}
      - REX_STATIC_DIR=/app/static
      - DATA_DIR=/app/data
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/healthz"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 10s

volumes:
  hub-data:
    driver: local
```

**关键设计决策：**
- 使用命名卷 `hub-data` 而非 bind mount，数据更安全
- 健康检查使用 `/healthz` 端点（已在 M1 实现）
- `build` 段可选：用户可以选择从源码构建或使用预构建镜像
- 环境变量通过 `.env` 文件传入（`REX_SECRET_KEY` 必需）
- `restart: unless-stopped` 保证异常退出后自动重启

### 2 Agent docker-compose.yaml

**功能目标：**
创建 `docker-compose.agent.yaml`，提供 Agent 的标准 docker-compose 部署配置。

**文件结构：**
```text
docker-compose.agent.yaml    新建
```

**配置设计：**
```yaml
services:
  agent:
    build:
      context: .
      dockerfile: Dockerfile.agent
    image: rexhub/agent:latest
    container_name: rex-agent
    restart: unless-stopped
    volumes:
      - agent-data:/app/data
      - /var/run/docker.sock:/var/run/docker.sock
    environment:
      - REX_SERVER=${REX_SERVER}
      - REX_TOKEN=${REX_TOKEN}
      - DATA_DIR=/app/data

volumes:
  agent-data:
    driver: local
```

**关键设计决策：**
- Docker.sock 挂载用于 Agent 扫描和管理 Docker 容器
- `agent-data` 命名卷持久化 Agent 身份（`agent.json`）
- 环境变量 `REX_SERVER` 和 `REX_TOKEN` 通过 `.env` 传入
- Agent 不需要端口暴露（主动出站连接 Hub）
