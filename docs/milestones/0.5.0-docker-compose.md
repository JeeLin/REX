## 子任务详细设计

### 1 Hub docker-compose.yaml

**功能目标：**
创建 `docker-compose.hub.yaml`，提供 Hub 的标准 docker-compose 部署配置。

**文件结构：**
```text
docker-compose.hub.yaml     新建
```

**配置设计：**
```yaml
services:
  hub:
    image: ghcr.io/${GITHUB_REPO_OWNER}/rex-hub:latest
    container_name: rex-hub
    restart: unless-stopped
    ports:
      - "3000:3000"
    volumes:
      - hub-data:/app/data
    environment:
      - REX_SECRET_KEY=${REX_SECRET_KEY}
      - REX_STATIC_DIR=/app/static
      - DATA_DIR=/app/data
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/healthz"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 10s

volumes:
  hub-data:
    driver: local
```

**关键设计决策：**
- 使用命名卷 `hub-data` 而非 bind mount，数据更安全
- 健康检查使用 `/healthz` 端点（已在 M1 实现）
- 环境变量通过 `.env` 文件传入（`REX_SECRET_KEY` 必需）
- `restart: unless-stopped` 保证异常退出后自动重启
- 镜像使用 GHCR（与 CI 一致）
- 需在 Dockerfile.hub 中加装 `curl`（debian:bookworm-slim 默认不含），使 healthcheck 可用

### 2 Agent docker-compose.yaml

**功能目标：**
创建 `docker-compose.agent.yaml`，提供 Agent 的标准 docker-compose 部署配置。

**文件结构：**
```text
docker-compose.agent.yaml    新建
```

**配置设计：**
```yaml
services:
  agent:
    image: ghcr.io/${GITHUB_REPO_OWNER}/rex-agent:latest
    container_name: rex-agent
    restart: unless-stopped
    volumes:
      - agent-data:/app/data
      - /var/run/docker.sock:/var/run/docker.sock
    environment:
      - REX_SERVER=${REX_SERVER}
      - REX_TOKEN=${REX_TOKEN}
      - DATA_DIR=/app/data

volumes:
  agent-data:
    driver: local
```

**关键设计决策：**
- Docker.sock 挂载用于 Agent 扫描和管理 Docker 容器
- `agent-data` 命名卷持久化 Agent 身份（`agent.json`）
- 环境变量 `REX_SERVER` 和 `REX_TOKEN` 通过 `.env` 传入
- Agent 不需要端口暴露（主动出站连接 Hub）

**Agent README 内容：**
```markdown
# REX Agent

轻量级反向代理进程，部署在内网服务器上，通过 TLS 加密隧道连接 REX Hub。

## 快速开始

### Docker
docker run -d \
  --name rex-agent \
  --restart unless-stopped \
  -e REX_SERVER=https://your-hub.com \
  -e REX_TOKEN=<注册令牌> \
  -v /var/run/docker.sock:/var/run/docker.sock \
  ghcr.io/rexhub/rex-agent:latest

### Docker Compose
参考 docker-compose.agent.yaml

### 二进制
curl -fsSL https://get.rexhub.dev/agent/linux-amd64 -o rex-agent
chmod +x rex-agent
./rex-agent --server https://your-hub.com --token <注册令牌> --name "内网 Agent"

## 环境变量

| 变量 | 必需 | 说明 |
|------|------|------|
| REX_SERVER | 是 | Hub 服务器地址（https://...） |
| REX_TOKEN | 是 | 环境注册令牌 |
| DATA_DIR | 否 | 数据目录（默认 /app/data） |

## 架构

Agent 从内网主动出站连接 Hub，建立 WebSocket 加密隧道。
内网服务器无需开放任何入站端口。
```

**CI Release job 变更：**
```yaml
# 旧：generate_release_notes: true
# 新：
- name: Read CHANGELOG
  id: changelog
  run: |
    # 提取当前版本的 changelog 内容
    VERSION=${GITHUB_REF#refs/tags/v}
    # 从 CHANGELOG.md 提取该版本段落
    ...

- name: Create GitHub Release
  uses: softprops/action-gh-release@v2
  with:
    body_path: release-body.md
    files: rex-hub-*.zip rex-agent-*.zip SHA256SUMS
```

**CI Agent Docker job 变更：**
在 build 步骤前添加复制 README 的步骤：
```yaml

```
