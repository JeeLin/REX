# 0.12.0: 代码质量清理与测试覆盖率提升

## Context

0.11.0 完成了 S3/MinIO 支持。随着项目功能不断累积，代码质量出现了一些技术债：
- 前端积累 146 个 ESLint 警告（vue/attributes-order、unused-vars 等）
- Rust 后端测试覆盖率仅 63.43%，多个核心模块低于 50%
- 无结构化的质量改进计划

版本类型：patch（纯修复/改进，不新增功能）

## 产品边界

**做什么：**
- 修复全部前端 ESLint 警告（146 个）
- 将 Rust 后端测试覆盖率从 63.43% 提升至 90%+
- 保持所有现有功能不变

**不做什么：**
- 不新增功能
- 不修改产品行为
- 不引入新的依赖
- 不重构架构

## 子任务详细设计

### 1 修复前端 ESLint 警告

**功能目标：**
消除全部 146 个 ESLint 警告，使 `bun run lint` 输出为 0 警告。

**修改文件：**
```text
packages/rex-console-web/src/
├── composables/useSidebar.ts               修改：移除未使用变量
├── features/agents/AgentConfigModal.vue     修改：vue/attributes-order
├── features/agents/AgentLogModal.vue        修改：vue/attributes-order
├── features/agents/AgentResetTokenModal.vue 修改：vue/attributes-order
├── features/agents/DeployGuide.vue          修改：vue/attributes-order
├── features/files/FileList.vue              修改：vue/attributes-order
├── features/files/TransferItem.vue          修改：vue/multiline-html-element-content-newline
├── features/s3/ObjectBrowser.vue            修改：vue/attributes-order
├── pages/Terminal.vue                       修改：vue/attributes-order + no-explicit-any
├── pages/Workspace.vue                      修改：vue/attributes-order + vue/v-on-event-hyphenation
└── 其他文件                                 修改：各类 vue/ 和 @typescript-eslint/ 规则
```

**策略：**
- 137 个可自动修复的警告使用 `bun run lint --fix` 修复
- 剩余不可自动修复的警告手动修复：
  - `@typescript-eslint/no-unused-vars`：移除未使用变量或加 `_` 前缀
  - `@typescript-eslint/no-explicit-any`：替换为具体类型
  - `vue/v-on-event-hyphenation`：事件名改为 kebab-case

**测试标准：**
- `bun run lint` 输出 0 errors, 0 warnings
- `bun run type-check` 通过
- `bun run build` 成功

**提交信息：**
```
fix: resolve all frontend lint warnings
```
