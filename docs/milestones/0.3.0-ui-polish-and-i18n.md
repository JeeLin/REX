# 0.3.0: UI 打磨与 i18n 补全

## Context

v0.2.0 完成了终端 SFTP 面板、全屏按钮、Settings 个人信息、审计统计 API、SQL 消息标签和标签关闭改进。所有核心功能模块（登录、仪表盘、环境管理、资源创建、工作空间、终端、SQL 控制台、文件管理、Agent 管理、审计日志、设置）均已实现。

1. **i18n 补全** — 填补硬编码中文字符串，确保所有用户可见文本支持中英文切换
2. **审计日志增强** — CSV 导出、时间范围筛选、操作类型筛选等 PRODUCT.md §3.10 定义但未实现的交互

版本类型：minor

## 子任务详细设计

### 1 ProfileSection i18n 化

**功能目标：**
将 `ProfileSection.vue` 中的 15+ 处硬编码中文字符串替换为 i18n 键，确保中英文切换正常工作。

**文件结构：**
```text
packages/rex-console-web/src/
├── features/settings/ProfileSection.vue    修改：i18n 化
├── i18n/zh.ts                              修改：添加 settings.profile.* 键
├── i18n/en.ts                              修改：添加 settings.profile.* 键
```

**接口设计：**
新增 i18n 键：
```text
settings.profile.title          → 个人信息 / Profile
settings.profile.username       → 用户名 / Username
settings.profile.edit           → 编辑 / Edit
settings.profile.password       → 密码 / Password
settings.profile.lastChanged    → 上次修改时间未知 / Last changed unknown
settings.profile.change         → 修改 / Change
settings.profile.editTitle      → 编辑用户名 / Edit Username
settings.profile.usernameLabel  → 用户名 / Username
settings.profile.usernamePlaceholder → 输入新用户名 / Enter new username
settings.profile.save           → 保存 / Save
settings.profile.cancel         → 取消 / Cancel
settings.profile.changeTitle    → 修改密码 / Change Password
settings.profile.currentPassword → 当前密码 / Current password
settings.profile.currentPasswordPlaceholder → 输入当前密码 / Enter current password
settings.profile.newPassword    → 新密码 / New password
settings.profile.newPasswordPlaceholder → 输入新密码（至少 6 位）/ Enter new password (min 6 chars)
settings.profile.saveSuccess    → 密码修改成功 / Password changed successfully
settings.profile.saveFailed     → 保存失败 / Save failed
settings.profile.passwordFailed → 密码修改失败 / Password change failed
```

**测试标准：**
- 中英文切换后所有文本正确显示
- 无硬编码中文残留

**提交信息：**
```
feat: add i18n support to profile settings section
```

### 2 审计日志筛选与导出

**功能目标：**
增强审计日志页面，添加 PRODUCT.md §3.10 定义的筛选和导出功能。

**文件结构：**
```text
packages/rex-console-web/src/
├── pages/AuditLog.vue                      修改：添加筛选栏和导出按钮
├── api/audit.ts                            修改：添加筛选参数和导出 API
├── i18n/zh.ts, en.ts                       修改：添加审计日志筛选 i18n 键
```

**接口设计：**
现有 API 已支持筛选参数（`from`, `to`, `type`），只需前端对接。

新增导出功能：
```text
GET /api/audit-log/export?from=&to=&type=    # CSV 导出（复用 list 客户端生成）
```

前端直接用现有 `listAuditLog` API 获取数据，在客户端生成 CSV 下载。

**交互设计：**
- 筛选栏：时间范围下拉（1h/24h/7d/30d）+ 操作类型下拉 + 重置按钮
- 统计卡片：总操作数 / 成功数 / 失败数
- 导出按钮：下载 CSV 文件

**测试标准：**
- 时间范围筛选正确过滤日志
- 操作类型筛选正确过滤日志
- CSV 导出包含筛选后的数据
- 重置按钮清除所有筛选条件

**提交信息：**
```
feat: add audit log filtering and CSV export
```
