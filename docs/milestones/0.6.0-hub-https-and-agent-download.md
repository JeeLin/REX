# 0.6.0: Hub HTTPS 支持与 Agent 下载

## Context

0.5.0 完成了 Docker 部署配置（Dockerfile、docker-compose、CI 调整）。当前 Hub 只支持 HTTP，不支持 TLS/HTTPS。PRODUCT.md 中描述的 Agent 二进制下载端点 `GET /api/agent/download` 尚未实现。

本里程碑解决两个核心问题：
1. Hub 原生支持 HTTPS，无需外部反向代理
2. Agent 可以从连接的 Hub 下载自身二进制，实现自更新和部署引导

版本类型：minor

## 产品边界

**做什么：**
- Hub 通过配置支持 TLS（证书 + 私钥），原生 HTTPS 监听
- Hub 提供 Agent 二进制下载端点（`GET /api/agent/download?os={os}&arch={arch}`）
- Agent 更新流程支持从 Hub 下载二进制（替代 GitHub Releases 直接下载）
- Docker 镜像支持 TLS 配置

**不做什么：**
- 不实现自动证书生成（Let's Encrypt / ACME）
- 不实现证书热加载（需重启 Hub）
- 不实现 Hub 自动从 GitHub 下载 Agent 二进制（手动放置或 CI 打包）
- 不修改 Agent 与 Hub 的 WebSocket 连接协议（已支持 wss://）

## 子任务详细设计

### 1 Hub TLS 配置与 HTTPS 监听

- **功能目标**
  Hub 可通过配置文件或 CLI 参数指定 TLS 证书和私钥路径，启用 HTTPS 监听。未配置时保持 HTTP 行为不变。

- **文件结构**（创建/修改哪些文件）
  - 修改：`crates/rex-hub/src/config.rs` — HubConfig 增加 tls_cert_path、tls_key_path 字段
  - 修改：`crates/rex-hub/src/bin/rex-hub.rs` — 根据配置选择 TcpListener 或 TcpListener::bind + TlsAcceptor
  - 修改：`crates/rex-common/src/cli.rs` — 增加 --tls-cert、--tls-key CLI 参数
  - 新增：`crates/rex-hub/src/tls.rs` — TLS 证书加载和管理

- **接口设计**
  配置文件（hub.yaml）：
  ```yaml
  listen: "0.0.0.0:3000"
  tls:
    cert: "/app/data/certs/hub.crt"
    key: "/app/data/certs/hub.key"
  ```

  环境变量：
  - `REX_TLS_CERT` — 证书文件路径
  - `REX_TLS_KEY` — 私钥文件路径

  CLI 参数：
  - `--tls-cert <path>` — 证书文件路径
  - `--tls-key <path>` — 私钥文件路径

  优先级：CLI > 环境变量 > 配置文件

- **交互设计**
  - 启动时检测 TLS 配置，日志输出 "TLS enabled, serving on https://0.0.0.0:3000"
  - 未配置 TLS 时日志输出 "HTTP only, serving on http://0.0.0.0:3000"
  - 证书加载失败时输出明确错误并退出

- **后端流程**
  1. 解析配置，检查 TLS 路径是否存在
  2. 加载证书和私钥（rustls 或 native-tls）
  3. 创建 TLS 配置（Protocol::TLS13 only）
  4. 包装 TcpListener 为 TlsAcceptor
  5. 使用 axum::serve 配合 TlsAcceptor 监听

- **测试标准**
  - 单元测试：配置解析（有/无 TLS）
  - 单元测试：证书加载（有效/无效路径）
  - 集成测试：HTTPS 请求成功响应
  - 集成测试：HTTP 请求在 TLS 模式下被拒绝

- **提交信息**
  `feat: add TLS/HTTPS support to Hub`

### 2 Agent 二进制下载端点

- **功能目标**
  Hub 提供 `GET /api/agent/download?os={os}&arch={arch}` 端点，返回对应平台的 Agent 二进制文件。支持版本匹配和签名校验。

- **文件结构**（创建/修改哪些文件）
  - 修改：`crates/rex-hub/src/routes.rs` — 添加 /api/agent/download 路由
  - 新增：`crates/rex-hub/src/agent_download.rs` — 下载端点实现
  - 修改：`crates/rex-hub/src/config.rs` — 添加 agent_binaries_dir 配置
  - 新增：`docs/milestones/0.6.0-reports/agent-binaries/` — Agent 二进制存放目录

- **接口设计**
  `GET /api/agent/download?os={os}&arch={arch}`
  - 查询参数：os（linux/darwin/windows）、arch（amd64/arm64/armv7l）
  - 返回：二进制文件流（application/octet-stream）
  - 响应头：Content-Disposition, X-Agent-Version, X-Agent-SHA256
  - 错误：404 如果指定平台无二进制

- **交互设计**
  - 前端部署指南中的下载按钮链接到此端点
  - 下载时显示进度和校验和
  - 支持断点续传（Content-Range）

- **后端流程**
  1. 接收请求，验证 os 和 arch 参数
  2. 查找对应的 Agent 二进制文件（agent-{os}-{arch}）
  3. 计算 SHA256（如果未缓存）
  4. 返回二进制流和元信息头

- **测试标准**
  - 单元测试：参数验证（有效/无效 os/arch）
  - 集成测试：下载成功返回二进制
  - 集成测试：平台不存在返回 404
  - 安全测试：路径遍历攻击防护

- **提交信息**
  `feat: add agent binary download endpoint`

### 3 Agent 更新流程集成 Hub 下载

- **功能目标**
  Agent 自更新流程支持从 Hub 下载二进制，而非直接从 GitHub Releases 下载。适用于内网环境无法直接访问 GitHub 的场景。

- **文件结构**（创建/修改哪些文件）
  - 修改：`crates/rex-agent/src/config.rs` — 添加 update_source 配置项
  - 修改：`crates/rex-agent/src/update.rs` — 支持从 Hub 下载
  - 修改：`crates/rex-common/src/updater.rs` — 添加 Hub 下载源

- **接口设计**
  Agent 配置（agent.yaml）：
  ```yaml
  update:
    source: hub  # hub | github，默认 github
  ```

  环境变量：
  - `REX_UPDATE_SOURCE=hub`

- **交互设计**
  - Agent 启动时检查更新源配置
  - 从 Hub 下载时使用 `GET /api/agent/download?os={os}&arch={arch}`
  - 下载后校验 SHA256 与 Hub 返回的 X-Agent-SHA256 一致

- **后端流程**
  1. Agent 检测到新版本可用
  2. 根据 update_source 选择下载源
  3. 如果是 hub：从 Hub 下载二进制
  4. 校验 SHA256
  5. 备份当前二进制，写入 update-state.json
  6. 正常退出，supervisor 替换

- **测试标准**
  - 单元测试：配置解析（update_source）
  - 集成测试：从 Hub 下载成功
  - 集成测试：SHA256 校验失败时拒绝更新

- **提交信息**
  `feat: support agent update from hub`

### 4 Docker 镜像 TLS 支持

- **功能目标**
  Docker 镜像支持 TLS 配置，通过挂载证书目录或环境变量启用 HTTPS。

- **文件结构**（创建/修改哪些文件）
  - 修改：`Dockerfile.hub` — 暴露 443 端口，添加证书目录
  - 修改：`docker-compose.yml` — 添加 TLS 相关环境变量和卷挂载
  - 新增：`docs/milestones/0.6.0-hub-https-and-agent-download.md` — 本文件

- **接口设计**
  Docker 环境变量：
  - `REX_TLS_CERT=/app/data/certs/hub.crt`
  - `REX_TLS_KEY=/app/data/certs/hub.key`

  卷挂载：
  - `./certs:/app/data/certs:ro`

- **交互设计**
  - 用户将证书文件放入 `./certs/` 目录
  - docker-compose.yml 中配置环境变量
  - Hub 启动时自动加载证书并启用 HTTPS

- **后端流程**
  1. Dockerfile 添加证书目录 `/app/data/certs`
  2. 暴露 443 端口（同时保留 3000 用于 HTTP）
  3. docker-compose.yml 添加 TLS 配置示例
  4. 更新部署文档

- **测试标准**
  - 文档测试：按照文档步骤可成功启用 HTTPS
  - 集成测试：Docker 容器启动后 HTTPS 可用

- **提交信息**
  `docs: add TLS support to Docker deployment`
