# 0.11.0: S3/MinIO 协议支持

## Context

0.10.0 完成了 SQLite 协议支持。当前已支持 SSH、SFTP、MySQL、PostgreSQL、Redis、Docker、SQLite 七种协议。PRODUCT.md 定义了 8 种协议，S3/MinIO 是最后一种未实现的协议。本里程碑实现 S3/MinIO 对象存储协议支持，让用户可以通过 REX Hub 管理 S3 兼容的对象存储服务。

版本类型：minor

## 产品边界

**做什么：**
- 新增 `rex-s3` crate，实现 S3/MinIO 对象存储连接和操作
- Hub 端 S3 WebSocket 会话管理
- 前端 S3 控制台（Bucket 列表、Object 浏览、上传/下载）
- S3 资源创建向导（Endpoint、Access Key、Secret Key、Bucket 表单）
- 工作空间 S3 面板集成
- S3 参与跨连接文件传输

**限制：**
- S3 连接通过 Agent 代理或直连（Endpoint 可为公网或内网地址）
- 仅支持单 Bucket 浏览操作（不支持跨 Bucket 管理）
- 不实现 Bucket 创建/删除（通过 S3 客户端工具管理）
- 上传/下载通过 WebSocket 通道，文件数据不经过浏览器

**不做什么：**
- 不实现 Bucket 创建/删除管理
- 不实现 S3 策略/ACL 管理
- 不实现跨 Bucket 批量迁移
- 不实现 S3 镜像同步/版本控制等高级功能
- 不实现 S3 定价分析

### 1 rex-s3 crate（S3Connector trait + S3 API 客户端）

- **功能目标**
  创建独立的 S3 协议 crate，实现 S3/MinIO 兼容的对象存储连接和操作。遵循现有 protocol crate 的 stub 模式。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-s3/Cargo.toml`
  - 新增：`crates/rex-s3/src/lib.rs` — 导出模块
  - 新增：`crates/rex-s3/src/connector.rs` — S3 连接器
  - 修改：`Cargo.toml`（workspace）— 添加 `rex-s3` 成员
  - 修改：`crates/rex-hub/Cargo.toml` — 添加 `rex-s3` 依赖

- **接口设计**

  S3 连接配置：
  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct S3Config {
      pub endpoint: String,              // S3 endpoint URL（如 https://s3.amazonaws.com 或 http://minio:9000）
      pub access_key: String,            // Access Key ID
      pub secret_key: String,            // Secret Access Key
      pub region: Option<String>,        // 区域（如 us-east-1）
      pub bucket: Option<String>,        // 默认 Bucket（可选，连接后可切换）
      pub force_path_style: bool,        // 强制 path-style（MinIO 兼容）
      pub name: Option<String>,          // 实例名称
  }
  ```

  Bucket 信息：
  ```rust
  #[derive(Debug, Serialize, Deserialize)]
  pub struct BucketInfo {
      pub name: String,
      pub creation_date: Option<String>,
  }
  ```

  Object 信息：
  ```rust
  #[derive(Debug, Serialize, Deserialize)]
  pub struct ObjectInfo {
      pub key: String,
      pub size: u64,
      pub last_modified: Option<String>,
      pub etag: Option<String>,
      pub content_type: Option<String>,
      pub storage_class: Option<String>,
      pub is_dir: bool,
  }
  ```

  S3Connector trait：
  ```rust
  #[async_trait]
  pub trait S3Connector: Send + Sync {
      async fn connect(&mut self) -> Result<()>;
      async fn list_buckets(&self) -> Result<Vec<BucketInfo>>;
      async fn list_objects(&self, bucket: &str, prefix: &str) -> Result<Vec<ObjectInfo>>;
      async fn get_object_info(&self, bucket: &str, key: &str) -> Result<ObjectInfo>;
      async fn upload_object(&self, bucket: &str, key: &str, data: Vec<u8>) -> Result<()>;
      async fn download_object(&self, bucket: &str, key: &str) -> Result<Vec<u8>>;
      async fn delete_object(&self, bucket: &str, key: &str) -> Result<()>;
      async fn close(&self) -> Result<()>;
  }
  ```

  S3ConnectorImpl 方法：
  ```rust
  impl S3ConnectorImpl {
      pub fn new(config: S3Config) -> Self { ... }
      pub fn from_json(json: &str) -> Result<Self> { ... }
      pub fn into_config(&self) -> S3Config { ... }
  }
  ```

- **后端流程**
  1. `connector.rs` 实现 `S3Connector`：
     - `connect()`：验证 endpoint 可达，尝试 ListBuckets 验证凭据
     - `list_buckets()`：调用 ListBuckets API，返回 BucketInfo 列表
     - `list_objects()`：调用 ListObjectsV2 API，返回 ObjectInfo 列表（含虚拟目录）
     - `get_object_info()`：调用 HeadObject API，返回单个对象元数据
     - `upload_object()`：调用 PutObject API 上传数据
     - `download_object()`：调用 GetObject API 下载数据
     - `delete_object()`：调用 DeleteObject API 删除对象
     - `close()`：关闭连接
  2. 与 rex-redis/rex-docker/rex-sqlite 一致，connector 是 stub 模式，实际连接通过 Agent 代理隧道

- **依赖选型**
  - `reqwest` — HTTP 客户端（复用 workspace 依赖，用于 S3 REST API）
  - `hmac` + `sha2` — AWS Signature V4 签名（sha2 已在 workspace，hmac 需新增到 workspace.dependencies）
  - `time` — 时间格式化（签名需要，workspace 已有依赖，无需新增）

- **测试标准**
  - 单元测试：S3Config 序列化/反序列化
  - 单元测试：S3Connector stub 行为
  - 单元测试：S3Connector trait object safety
  - 单元测试：BucketInfo / ObjectInfo 序列化

### 2 Hub S3 WebSocket 会话管理

- **功能目标**
  Hub 端实现 S3 会话的 WebSocket 通道管理，支持前端通过 WebSocket 发送 S3 操作命令并接收响应。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-hub/src/ws_s3.rs` — S3 WebSocket 会话管理
  - 修改：`crates/rex-hub/src/lib.rs` — 注册 `ws_s3` 模块
  - 修改：`crates/rex-hub/src/routes.rs` — 添加 S3 WebSocket 路由

- **接口设计**

  WebSocket 消息协议（S3 频道）：
  ```text
  客户端 → Hub：
  { "type": "command", "id": "uuid", "action": "buckets", "params": {} }
  { "type": "command", "id": "uuid", "action": "objects", "params": { "bucket": "my-bucket", "prefix": "" } }
  { "type": "command", "id": "uuid", "action": "info", "params": { "bucket": "my-bucket", "key": "path/to/file.txt" } }
  { "type": "command", "id": "uuid", "action": "upload", "params": { "bucket": "my-bucket", "key": "path/to/file.txt", "data": "base64..." } }
  { "type": "command", "id": "uuid", "action": "download", "params": { "bucket": "my-bucket", "key": "path/to/file.txt" } }
  { "type": "command", "id": "uuid", "action": "delete", "params": { "bucket": "my-bucket", "key": "path/to/file.txt" } }
  { "type": "ping" }

  Hub → 客户端：
  { "type": "response", "id": "uuid", "data": [...] }
  { "type": "error", "id": "uuid", "message": "bucket not found" }
  { "type": "pong" }
  { "type": "connected", "server": { "endpoint": "https://s3.amazonaws.com", "buckets": 5 } }
  { "type": "disconnected", "reason": "connection lost" }
  ```

  路由：
  ```text
  WebSocket /ws/s3/:resource_id
  ```

- **后端流程**
  1. 前端发起 WebSocket 连接到 `/ws/s3/:resource_id`
  2. Hub 从数据库查找 resource 配置，获取 S3 连接信息
  3. Hub 建立到 S3 endpoint 的连接（通过 Agent 代理或直连）
  4. 返回 `connected` 消息（含 endpoint 和 bucket 数量）
  5. 前端发送 `command` 消息 → Hub 执行 S3 操作 → 返回 `response`
  6. 上传/下载操作使用 base64 编码通过 WebSocket 传输（文件数据不经过浏览器中转）
  7. 连接断开时返回 `disconnected` 消息

- **测试标准**
  - 单元测试：S3 WebSocket 消息序列化/反序列化
  - 单元测试：路由注册正确

- **提交信息**
  `feat: add S3 WebSocket session management in Hub`

  ```
  ┌──────────────────────────────────────────────────┐
  │ ● S3 · my-minio · http://minio:9000              │  ← 顶部状态栏
  ├──────────────────────────────────────────────────┤
  │ 📦 Buckets                                       │  ← Bucket 列表区域
  │ ┌────────────────────────────────────────────┐   │
  │ │ 🪣 my-bucket          2024-01-15           │   │
  │ │ 🪣 backups            2024-01-10           │   │
  │ │ 🪣 public-assets      2024-01-08           │   │
  │ └────────────────────────────────────────────┘   │
  ├──────────────────────────────────────────────────┤
  │ 📁 my-bucket / uploads/                    🔄    │  ← 面包屑路径
  ├──────────────────────────────────────────────────┤
  │ 名称                  大小      修改时间      操作 │  ← Object 表格头
  ```
