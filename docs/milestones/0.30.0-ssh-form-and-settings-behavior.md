# 0.30.0: SSH 表单补全与设置行为对接

## Context

0.29.0 完成了环境编辑/删除、资源探活、右键菜单 action、i18n 同步等。但 SSH 资源创建表单（密钥上传、编码选择、保活间隔）和设置页大部分选项的实际行为仍缺失。本版本聚焦将这两个"半成品"打磨到完整可用。

版本类型：minor（新功能：SSH 密钥上传、设置行为对接、仪表盘统计）

## 产品边界

**做什么：**
- SSH 密钥文件拖拽/点击上传（后端接收 + 存储）
- SSH 终端编码选择（UTF-8/GBK/ISO-8859-1）
- SSH 保活间隔选择（30s/60s/120s）
- 设置页各选项实际行为对接后端
- 仪表盘统计卡片对接真实 API

**不做什么：**
- Redis 真实连接实现
- 新协议支持
- 多用户/RBAC

## 子任务详细设计

### 1 SSH 密钥上传后端接口

- **功能目标**：接收 SSH 密钥文件，安全存储到 Hub 数据目录
- **文件结构**：
  - 后端：`crates/rex-hub/src/routes.rs`（新增 `POST /api/resources/:id/ssh-key` 上传端点）
  - 后端：`crates/rex-hub/src/resource.rs`（新增 `upload_ssh_key` 函数）
- **接口设计**：
  - `POST /api/environments/:env_id/resources/:id/ssh-key`
  - Content-Type: multipart/form-data
  - Body: `file` (PEM/PPK/OpenSSH 格式)
  - Response: `{ "key_path": "/data/keys/{resource_id}.key" }`
- **后端流程**：
  1. 验证文件大小 ≤ 64KB
  2. 读取文件头判断格式（PEM: `-----BEGIN`，PPK: `PuTTY-User-Key-File`，OpenSSH: `ssh-`）
  3. 写入 `{data_dir}/keys/{resource_id}.key`，权限 0600
  4. 返回存储路径（实际路径不返回给前端，前端保存 resource_id 即可）
  5. 审计日志记录上传操作
- **测试标准**：
  - 上传合法 PEM 密钥 → 200，key_path 非空
  - 上传超过 64KB 文件 → 413
  - 上传非法格式 → 400
  - 同一资源再次上传 → 覆盖旧文件
- **提交信息**：`feat: add SSH key file upload endpoint`

### 2 SSH 密钥上传前端 + 编码/保活选项

- **功能目标**：ResourceNew SSH 步骤支持拖拽上传密钥，添加编码和保活选项
- **文件结构**：
  - 前端：`packages/rex-console-web/src/pages/ResourceNew.vue`（修改 SSH 表单）
- **交互设计**（参考 PRODUCT.md 3.4）：
  - 密钥模式：拖放/点击上传区域，支持 PEM、PPK、OpenSSH 格式
  - 上传成功后显示文件名和大小
  - 终端编码下拉框：UTF-8（推荐）/ GBK / ISO-8859-1
  - 保活间隔下拉框：60s（默认）/ 30s / 120s
  - `buildConfigJson()` 增加 `encoding` 和 `keepalive_interval` 字段
- **测试标准**：
  - 拖拽上传密钥文件 → 显示文件名
  - 选择编码和保活 → 保存到 config_json
  - type-check 通过
- **提交信息**：`feat: add SSH key upload UI and encoding/keepalive options`

### 3 设置页行为对接

- **功能目标**：设置页各选项实际对接后端行为
- **文件结构**：
  - 后端：`crates/rex-hub/src/routes.rs`（增强 settings API）
  - 前端：`packages/rex-console-web/src/pages/Settings.vue`（修改）
- **当前状态**：
  - 设置页有基本布局但大多数 toggle 只是 UI 状态，未实际影响后端行为
- **需要对接的设置项**：

| 设置项 | 后端行为 |
|--------|---------|
| 终端字体大小 | 存储到 localStorage（纯前端） |
| 终端字体族 | 存储到 localStorage（纯前端） |
| 光标闪烁 | 存储到 localStorage（纯前端） |
| 保活间隔 | 存储到 settings 表，SSH 连接时使用 |
| 会话超时 | 存储到 settings 表，auth middleware 使用 |
| 配置加密 | 已对接（AES-256-GCM） |
| 审计日志开关 | 存储到 settings 表，控制审计写入 |
| 主题 | 已对接（localStorage） |
| 语言 | 已对接（localStorage） |
| 自动更新检查 | 存储到 settings 表，控制检查频率 |

- **测试标准**：
  - 修改字体大小 → 终端实际生效
  - 修改保活间隔 → SSH 连接使用新值
  - 修改会话超时 → 超时后需重新认证
  - 审计日志关闭 → 不再写入审计记录
- **提交信息**：`feat: wire settings page behaviors to backend`

### 4 仪表盘统计 API 对接

- **功能目标**：Dashboard 4 个统计卡片显示真实数据
- **文件结构**：
  - 后端：`crates/rex-hub/src/routes.rs`（新增 `GET /api/dashboard/stats` 端点）
  - 前端：`packages/rex-console-web/src/pages/Dashboard.vue`（修改）
- **接口设计**：
  - `GET /api/dashboard/stats`
  - Response:
    ```json
    {
      "environment_count": 3,
      "resource_count": 12,
      "agent_online_count": 2,
      "today_operations": 47
    }
    ```
- **后端流程**：
  - `environment_count`: `SELECT COUNT(*) FROM environments`
  - `resource_count`: `SELECT COUNT(*) FROM resources`
  - `agent_online_count`: `SELECT COUNT(*) FROM agents WHERE status = 'online'`
  - `today_operations`: `SELECT COUNT(*) FROM audit_logs WHERE created_at >= date('now')`
- **测试标准**：
  - 返回正确的统计数据
  - 无数据时返回 0
- **提交信息**：`feat: add dashboard stats API with real data`
