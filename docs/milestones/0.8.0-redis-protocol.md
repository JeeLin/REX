# 0.8.0: Redis 协议支持

## Context

0.7.0 完成了 ACME 自动证书。当前已支持 SSH、SFTP、MySQL、PostgreSQL 四种协议。PRODUCT.md 定义了 8 种协议，其中 Redis、Docker、SQLite、S3/MinIO 尚未实现。本里程碑实现 Redis 协议支持，为用户提供缓存管理能力。

版本类型：minor

## 产品边界

**做什么：**
- 新增 `rex-redis` crate，实现 Redis RESP 协议解析和命令执行
- Hub 端 Redis WebSocket 会话管理
- 前端 Redis 命令控制台（命令输入 + 结果展示 + 历史记录）
- Redis 资源创建向导（连接配置表单）
- 工作空间 Redis 面板集成

**限制：**
- Redis 连接通过 Agent 代理或直连
- 不实现 Redis Cluster / Sentinel 支持（单实例模式）
- 不实现 Pub/Sub 订阅面板（仅命令行交互）

**不做什么：**
- 不实现 Docker、SQLite、S3/MinIO 协议（后续里程碑）
- 不实现 Redis Cluster 集群管理
- 不实现数据可视化 / 监控面板

## 子任务详细设计

### 1 rex-redis crate（RESP 协议 + RedisConnector trait）

- **功能目标**
  创建独立的 Redis 协议 crate，实现 RESP（Redis Serialization Protocol）解析和 Redis 命令执行接口。遵循现有 protocol crate 的 stub 模式。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-redis/Cargo.toml`
  - 新增：`crates/rex-redis/src/lib.rs` — 导出模块
  - 新增：`crates/rex-redis/src/connector.rs` — Redis 连接器和 RESP 解析
  - 新增：`crates/rex-redis/src/resp.rs` — RESP 协议编解码
  - 修改：`Cargo.toml`（workspace）— 添加 `rex-redis` 成员和 `tokio-util` 依赖
  - 修改：`crates/rex-hub/Cargo.toml` — 添加 `rex-redis` 依赖

- **接口设计**

  Redis 连接配置：
  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct RedisConfig {
      pub host: String,
      pub port: u16,
      pub password: Option<String>,
      pub db: u8,
      pub name: Option<String>,
  }
  ```

  Redis 响应类型（RESP）：
  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  #[serde(tag = "type", content = "value")]
  pub enum RedisValue {
      Status(String),      // +OK
      Error(String),       // -ERR message
      Integer(i64),        // :1000
      Bulk(String),        // $6\r\nfoobar
      Array(Vec<RedisValue>), // *2\r\n$3\r\nfoo\r\n$3\r\nbar
      Null,                // $-1
  }

  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct RedisResponse {
      pub value: RedisValue,
      pub elapsed_ms: u64,
  }
  ```

  RedisConnector trait：
  ```rust
  #[async_trait]
  pub trait RedisConnector: Send + Sync {
      async fn connect(&mut self) -> Result<()>;
      async fn execute(&self, command: &str) -> Result<RedisResponse>;
      async fn info(&self) -> Result<HashMap<String, String>>;
      async fn close(&self) -> Result<()>;
  }
  ```

- **后端流程**
  1. `resp.rs` 实现 RESP 协议编解码（读取字节流 → 解析为 `RedisValue`，编码命令为 RESP 格式）
  2. `connector.rs` 实现 `RedisConnector`：
     - `connect()`：通过 TCP 连接 Redis 服务器，发送 AUTH（如果有密码），发送 SELECT（如果 db != 0）
     - `execute()`：发送命令字符串，解析 RESP 响应，返回 `RedisResponse`
     - `info()`：执行 `INFO server`，解析为 key-value map
     - `close()`：关闭 TCP 连接
  3. 与 rex-mysql 一致，connector 是 stub 模式，实际连接通过 Agent 代理隧道

- **依赖选型**
  - `tokio-util` — 编解码器框架（`Framed` + `Decoder`/`Encoder`）
  - 无需额外 Redis 客户端库，手动实现 RESP 协议（保持轻量，与 stub 模式一致）

- **测试标准**
  - 单元测试：RESP 编解码（Status、Error、Integer、Bulk、Array、Null）
  - 单元测试：RedisConfig 序列化/反序列化
  - 单元测试：RedisConnector stub 行为（connect/execute/close）
  - 单元测试：RedisConnector trait object safety

- **提交信息**
  `feat: add rex-redis crate with RESP protocol and RedisConnector trait`

### 2 Hub Redis WebSocket 会话管理

- **功能目标**
  Hub 端实现 Redis 会话的 WebSocket 通道管理，支持前端通过 WebSocket 发送 Redis 命令并接收响应。

- **文件结构**（创建/修改哪些文件）
  - 新增：`crates/rex-hub/src/ws_redis.rs` — Redis WebSocket 会话管理
  - 修改：`crates/rex-hub/src/lib.rs` — 注册 `ws_redis` 模块
  - 修改：`crates/rex-hub/src/routes.rs` — 添加 Redis WebSocket 路由

- **接口设计**

  WebSocket 消息协议（Redis 频道）：
  ```text
  客户端 → Hub：
  { "type": "command", "id": "uuid", "command": "GET mykey" }
  { "type": "ping" }

  Hub → 客户端：
  { "type": "response", "id": "uuid", "value": { "type": "Bulk", "value": "hello" }, "elapsed_ms": 2 }
  { "type": "error", "id": "uuid", "message": "ERR wrong number of arguments" }
  { "type": "pong" }
  { "type": "connected", "server": { "redis_version": "7.0.0", "mode": "standalone" } }
  { "type": "disconnected", "reason": "connection lost" }
  ```

  路由：
  ```text
  WebSocket /ws/redis/{resource_id}
  ```

- **后端流程**
  1. 前端发起 WebSocket 连接到 `/ws/redis/{resource_id}`
  2. Hub 从数据库查找 resource 配置，获取 Redis 连接信息
  3. Hub 建立到 Redis 的 TCP 连接（通过 Agent 代理或直连）
  4. 返回 `connected` 消息（含服务器信息）
  5. 前端发送 `command` 消息 → Hub 通过 TCP 发送到 Redis → 解析 RESP 响应 → 返回 `response`
  6. 连接断开时返回 `disconnected` 消息

- **测试标准**
  - 单元测试：Redis WebSocket 消息序列化/反序列化
  - 单元测试：路由注册正确

- **提交信息**
  `feat: add Redis WebSocket session management in Hub`

- **功能目标**
  实现 Redis 命令控制台前端组件，支持命令输入、结果展示、历史记录。参考原型 `sql.html` 的交互模式。

- **文件结构**（创建/修改哪些文件）
  - 新增：`packages/rex-console-web/src/features/redis/RedisConsole.vue` — Redis 命令控制台主组件
  - 新增：`packages/rex-console-web/src/features/redis/RedisResult.vue` — 结果展示组件
  - 新增：`packages/rex-console-web/src/features/redis/RedisHistory.vue` — 历史记录面板
  - 新增：`packages/rex-console-web/src/features/redis/useRedisSession.ts` — WebSocket 会话管理
  - 新增：`packages/rex-console-web/src/api/redis.ts` — Redis API 客户端
  - 修改：`packages/rex-console-web/src/i18n/zh.ts` — 添加 Redis 相关翻译
  - 修改：`packages/rex-console-web/src/i18n/en.ts` — 添加 Redis 相关翻译

- **接口设计**

  RedisConsole props：
  ```typescript
  interface Props {
    resourceId: string
    resourceName: string
  }
  ```

  useRedisSession composable：
  ```typescript
  function useRedisSession(resourceId: Ref<string>) {
    return {
      connected: Ref<boolean>,
      serverInfo: Ref<Record<string, string> | null>,
      connect: () => Promise<void>,
      disconnect: () => void,
      execute: (command: string) => Promise<RedisResponse>,
    }
  }
  ```

- **交互设计**（参考原型 sql.html）

  Redis 控制台布局：
  ```
  ┌──────────────────────────────────────────┐
  │ ● Redis · my-redis · connected            │  ← 顶部状态栏
  ├──────────────────────────────────────────┤
  │ > SET greeting "hello world"              │  ← 命令输入行
  │ OK                                        │
  │ > GET greeting                            │
  │ "hello world"                             │
  │ > HSET user:1 name "Alice" age 30         │
  │ (integer) 2                               │
  │ > HGETALL user:1                          │
  │ 1) "name"                                 │
  │ 2) "Alice"                                │
  │ 3) "age"                                  │
  │ 4) "30"                                   │
  ├──────────────────────────────────────────┤
  │ > _                                       │  ← 底部输入框（带闪烁光标）
  └──────────────────────────────────────────┘
  ```

  交互细节：
  - 命令输入：底部 monospace 输入框，Enter 发送
  - 结果展示：终端风格，按 RESP 类型着色（Status=绿色、Error=红色、Integer=橙色、Bulk=白色、Array=缩进列表）
  - 历史记录：↑↓ 键切换历史命令
  - 命令补全：Tab 补全 Redis 命令名（GET、SET、HGETALL 等常用命令）
  - 清屏：`Ctrl+L` 清空显示
  - 连接断开：显示断开提示，支持重连按钮

- **后端流程**
  无（纯前端组件，通过 WebSocket 与 Hub 通信）

- **测试标准**
  - 组件渲染测试：RedisConsole 渲染成功
  - composable 测试：useRedisSession 状态管理

- **提交信息**
  `feat: add Redis command console frontend`
