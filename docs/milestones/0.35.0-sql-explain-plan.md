# 0.35.0: SQL 执行计划

## Context

0.34.0 完成了移动端体验完善。PRODUCT.md 要求 SQL 控制台结果区包含三个标签页：结果、消息、执行计划。目前只有前两个，执行计划功能缺失。

版本类型：minor（新功能）

## 产品边界

**做什么：**
- 后端 SQL 执行计划 API（EXPLAIN 支持 MySQL 和 PostgreSQL）
- 前端执行计划标签页（显示 EXPLAIN 结果表格）
- 结果区标签栏增加「执行计划」标签

**不做什么：**
- 可视化执行计划树形图（纯表格展示即可）
- 修改已有查询执行流程
- 新增其他 SQL 诊断功能

## 子任务详细设计

### 1 后端 SqlConnector trait 新增 explain 方法

- **功能目标**：在 `rex-common` 的 `SqlConnector` trait 中添加 `explain` 方法，返回通用的执行计划结果
- **文件结构**：修改 `crates/rex-common/src/sql.rs`
- **接口设计**：
  ```rust
  // 新增数据类型 — 使用通用列式结构适配不同方言
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct ExplainResult {
      pub columns: Vec<String>,
      pub rows: Vec<Vec<serde_json::Value>>,
      pub raw_output: String,
  }
  
  // trait 新增方法
  async fn explain(&self, sql: &str) -> Result<ExplainResult>;
  ```
  - `columns` + `rows`：结构化的列式数据（MySQL EXPLAIN 直接映射，PostgreSQL 解析后适配）
  - `raw_output`：原始文本输出，作为 fallback 展示
- **测试标准**：ExplainRow/ExplainResult 序列化反序列化测试
- **提交信息**：`feat: add ExplainResult type and explain method to SqlConnector trait`

### 2 后端 MySQL/PostgreSQL 连接器实现 explain

- **功能目标**：MySQL 执行 `EXPLAIN {sql}`，PostgreSQL 执行 `EXPLAIN (FORMAT JSON) {sql}`，解析结果为 `ExplainResult`
- **文件结构**：修改 `crates/rex-mysql/src/connector.rs`，修改 `crates/rex-postgresql/src/connector.rs`
- **接口设计**：实现 `explain` 方法
  - MySQL：执行 `EXPLAIN {sql}`，解析标准 EXPLAIN 输出为 ExplainResult
  - PostgreSQL：执行 `EXPLAIN (FORMAT JSON) {sql}`，解析 JSON 为 ExplainResult，保留原始文本于 raw_output
- **交互设计**：无
- **后端流程**：
  1. 连接数据库
  2. 构造 EXPLAIN 语句（根据方言不同）
  3. 执行并解析结果
  4. 返回 ExplainResult
- **测试标准**：连接器级别单元测试（mock 连接或测试数据库）
- **提交信息**：`feat: implement explain for MySQL and PostgreSQL connectors`

### 3 后端 EXPLAIN API 端点

- **功能目标**：新增 `POST /api/resources/:resource_id/sql/explain` 端点
- **文件结构**：修改 `crates/rex-hub/src/sql.rs`
- **接口设计**：
  - 请求：`{ sql: string }`（与 execute 共用 `ExecuteRequest`）
  - 响应：`ApiResponse<ExplainResult>`
- **后端流程**：
  1. 获取 SqlConnector
  2. 调用 `connector.explain(&input.sql)`
  3. 返回结果
- **测试标准**：API 测试（未知资源返回 404，空 SQL 返回 400）
- **提交信息**：`feat: add POST /sql/explain API endpoint`

### 4 前端 API + 执行计划标签页

- **功能目标**：前端调用 EXPLAIN API，在 SqlResults.vue 增加「执行计划」标签页
- **文件结构**：
  - 修改 `packages/rex-console-web/src/api/sql.ts`（新增 `explainSql` 函数）
  - 修改 `packages/rex-console-web/src/features/sql/SqlResults.vue`（增加执行计划标签页）
  - 修改 `packages/rex-console-web/src/features/sql/useSqlTabActions.ts`（执行时同时获取执行计划）
- **接口设计**：
  ```typescript
  // api/sql.ts
  export interface ExplainResult {
    columns: string[]
    rows: unknown[][]
    raw_output: string
  }
  
  export function explainSql(resourceId: string, sql: string): Promise<ExplainResult>
  ```
- **交互设计**：
  - 结果区标签栏：结果 | 消息 | 执行计划
  - 执行计划标签页：表格展示 EXPLAIN 结果
  - 执行 SQL 时自动请求 EXPLAIN（可选，或点击标签时懒加载）
- **后端流程**：无
- **测试标准**：TypeScript 类型检查通过，UI 标签切换正常
- **提交信息**：`feat: add explain tab to SQL results panel`
