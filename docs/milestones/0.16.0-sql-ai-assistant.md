# 0.16.0: SQL AI 助手

## Context

0.15.0 完成了跨连接文件传输（执行引擎 + 连接器解析 + API 集成）。PRODUCT.md 描述了 SQL AI 助手面板（Ctrl+Shift+A），提供对话式交互、上下文感知、快捷操作和代码复制功能。目前代码库中无任何 AI/LLM 相关实现。

版本类型：minor（新功能）

## 产品边界

**做什么：**
- 后端：LLM 代理 API（POST /api/ai/chat），支持 SSE 流式响应，兼容 OpenAI Chat Completions 格式
- 后端：AI 配置管理（provider、api_key、model、base_url），存储在数据库中
- 前端：AI 助手抽屉面板（右侧滑出），对话式交互，流式显示 AI 回复
- 前端：快捷操作按钮（生成 SQL、分析慢查询、表关系）
- 前端：上下文栏（当前数据库、表、查询）
- 前端：代码操作（复制 AI 生成的 SQL 到编辑器）
- 前端：键盘快捷键 Ctrl+Shift+A 切换面板

**不做什么：**
- 不实现多轮对话历史持久化（内存中保存，页面刷新清空）
- 不实现 SQL 自动执行（AI 生成的 SQL 只复制，不自动运行）
- 不实现向量检索/RAG
- 不内置特定 AI provider SDK（使用通用 HTTP 调用）
- 不实现设置页面中的 AI 配置 UI（通过 API 直接配置，后续里程碑补充）

## 子任务详细设计

### 1 AI 配置存储 + 代理 API（SSE 流式）

**功能目标：**
后端实现 LLM 代理 API，接收用户消息和 SQL 上下文，转发到配置的 AI provider，通过 SSE 流式返回响应。AI 配置（provider、api_key、model、base_url）存储在数据库中。

**文件结构：**
```text
crates/rex-hub/src/
├── ai.rs                新增：AI 配置管理 + 代理 API handler
├── routes.rs            修改：注册 /api/ai/* 路由
crates/rex-common/src/
├── db.rs                修改：新增 ai_config 表
```

**数据模型：**
```sql
CREATE TABLE ai_config (
  id TEXT PRIMARY KEY DEFAULT 'default',
  provider TEXT NOT NULL DEFAULT 'openai',
  api_key TEXT NOT NULL DEFAULT '',  -- AES-256 加密存储，与资源密码一致
  model TEXT NOT NULL DEFAULT 'gpt-4o',
  base_url TEXT NOT NULL DEFAULT 'https://api.openai.com/v1',
  updated_at TEXT NOT NULL
);
```
数据库迁移：在现有 migration 模式中新增 `ai_config` 表（复用 `db.rs` 的 `ensure_schema` 模式）。API key 使用 `SshResourceConfig::encrypt_sensitive` 相同的 AES-256 加密方案，写入时加密、读取时解密。

**API 设计：**
```text
POST /api/ai/chat
Body: {
  "messages": [
    { "role": "user", "content": "请优化这个慢查询: SELECT * FROM users WHERE..." },
    { "role": "assistant", "content": "建议添加索引..." },
    { "role": "user", "content": "能给我具体的 CREATE INDEX 语句吗？" }
  ],
  "context": {
    "database": "mydb",
    "tables": ["users", "orders"],
    "dialect": "mysql"
  }
}
Response: SSE stream (text/event-stream)
  data: {"type":"token","content":"CREATE"}
  data: {"type":"token","content":" INDEX"}
  data: {"type":"done","content":"完整回复内容"}
  data: {"type":"error","content":"API key 无效"}

GET /api/ai/config
Response: { "data": { "provider":"openai","model":"gpt-4o","configured":true } }

PUT /api/ai/config
Body: { "provider":"openai","api_key":"sk-xxx","model":"gpt-4o","base_url":"https://api.openai.com/v1" }
Response: 200 OK
```

**后端流程：**
1. 接收 chat 请求，验证消息列表非空
2. 从数据库读取 AI 配置
3. 构建 OpenAI 兼容的请求体（messages + system prompt + stream=true）
4. 使用 reqwest 以 SSE 方式流式请求 AI provider
5. 将每个 token 转发给客户端（SSE 格式）
6. 收集完整回复，返回 `type:done`

**System Prompt：**
```
你是一个 SQL 专家助手。根据用户提供的 SQL 上下文（数据库类型、表结构、当前查询），
帮助用户优化 SQL、分析问题、生成查询。

规则：
1. 回复中如果包含 SQL 代码，使用 ```sql 代码块包裹
2. 解释要简洁明了
3. 如果用户没有提供足够的上下文，主动询问
```

**测试标准：**
- 读取/更新 AI 配置 API 正常工作
- chat 请求在没有配置时返回 400
- SSE 流式响应格式正确
- 无效 API key 时返回错误事件

**useAiChat composable：**
```typescript
function useAiChat(context: Ref<SqlContext>) {
  const messages = ref<AiMessage[]>([])
  const isStreaming = ref(false)

  async function send(content: string) {
    // 添加用户消息
    messages.value.push({ id: Date.now().toString(), role: 'user', content })
    
    // 创建空的 AI 消息占位
    const aiMsg = reactive({ id: Date.now().toString(), role: 'assistant', content: '', streaming: true })
    messages.value.push(aiMsg)

    // SSE 流式接收
    const response = await fetch('/api/ai/chat', {
      method: 'POST',
      body: JSON.stringify({ messages: messages.value.slice(0, -1), context: context.value })
    })
    
    const reader = response.body!.getReader()
    const decoder = new TextDecoder()
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      const text = decoder.decode(value)
      // 解析 SSE 事件，更新 aiMsg.content
    }
    aiMsg.streaming = false
  }

  return { messages, isStreaming, send }
}
```

- 流式输出时显示光标动画
- 上下文栏自动读取当前 SQL 标签的数据库/表信息
