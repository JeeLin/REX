# 0.21.0: Agent 自动更新流程打通

## Context

0.20.0 完成了数据连接器的真实实现（SQLite、S3、sysinfo）。此前 0.6.0-0.7.0 阶段已实现 Hub 侧的完整更新基础设施（检查、下载、supervisor 替换、回滚），Agent 心跳中 Hub 也会返回 `needs_update` 标志。但 Agent 侧从未响应这个标志——收到 `needs_update: true` 后什么都不做，自动更新流程断在 Agent 端。

本里程碑打通 Agent 自动更新的完整链路：Agent 心跳响应 → 下载二进制 → 状态写入 → supervisor 替换 → 前端开关对接。

版本类型：minor（新功能，向后兼容），版本号 0.21.0。

## 产品边界

**做什么：**
- Agent 收到 `needs_update` 后自动下载新版二进制并触发 supervisor 更新
- Agent 新增 `auto_update` 配置项，控制是否自动更新
- 前端 Agent 配置弹窗的"自动更新"开关对接后端 API
- 前端设置页版本总览显示 Agent 版本和更新状态

**不做什么：**
- 不修改 Hub 自身的更新流程（已完成）
- 不新增 Agent 日志收集功能
- 不实现 Agent 配置推送（热更新）
- 不实现 SSH 密钥管理
- 不修改 WebSocket 协议（现有 auth/heartbeat/heartbeat_ack 足够）

## 子任务详细设计

### 1 Agent 侧自动更新逻辑

**功能目标：**
在 Agent WebSocket 客户端中响应 `needs_update` 标志，执行下载、备份、状态写入、退出流程，由 supervisor 完成二进制替换。

**文件结构：**
```text
crates/
├── rex-agent/
│   └── src/
│       ├── ws.rs         修改：处理 heartbeat_ack 中的 needs_update
│       └── bin/
│           └── rex-agent.rs  修改：使用 run_update_supervisor
├── rex-common/
│   └── src/
│       └── supervisor.rs  无需修改（已有完整实现）
```

**接口设计：**
Agent WebSocket 收到 `heartbeat_ack` 消息时，解析 `needs_update` 字段：
```json
{
  "msg_type": "heartbeat_ack",
  "payload": {
    "hub_version": "v0.21.0",
    "needs_update": true
  }
}
```

**后端流程：**
1. Agent `ws.rs` 的 `heartbeat_ack` 匹配分支中：
   - 解析 `needs_update` 和 `hub_version`
   - 如果 `needs_update == true` 且 `config.update.auto_update == true`：
     - 调用 `UpdateChecker::download_from_hub()` 下载新二进制到 staging 目录
     - 校验 SHA256：Hub 下载响应头 `X-Agent-SHA256` 与实际计算值对比，不匹配则中止
     - 调用 `UpdateChecker::backup_current()` 备份当前二进制
     - 写入 `update-state.json`（phase=Requested, target_version, staged_path, rollback_path）
     - 以 exit code 10 退出（`std::process::exit(10)`）
2. `rex-agent.rs` 入口改为使用 `run_update_supervisor()` 替代简单的 supervisor 循环


### 2 Agent auto_update 配置项 + Hub API 支持

**功能目标：**
Agent 配置新增 `auto_update` 布尔开关（默认 true）。Hub 侧新增 API 支持查看和修改 Agent 的 auto_update 设置。

**文件结构：**
```text
crates/
├── rex-agent/
│   └── src/
│       ├── config.rs     修改：新增 auto_update 字段
│       └── ws.rs         修改：心跳 payload 新增 auto_update 字段
├── rex-hub/
│   └── src/
│       ├── migrations.sql  修改：agents 表新增 config_json 列
│       ├── routes.rs     修改：新增 Agent 配置 API 路由
│       ├── agent.rs      修改：新增 get/set agent config 处理函数
│       └── ws.rs         修改：心跳处理解析并存储 auto_update
```

**接口设计：**

`GET /api/agents/:agent_id/config` → 返回 Agent 配置（仅 auto_update 等可读字段）
```json
{
  "data": {
    "auto_update": true
  }
}
```

`PATCH /api/agents/:agent_id/config` → 修改 Agent 配置
```json
{
  "auto_update": false
}
```

Agent 配置存储在 `agents` 表的 `config_json` 列（JSON 字段），与资源的 config_json 复用同一模式。

Agent WebSocket 心跳时将 `auto_update` 状态上报给 Hub：
```json
{
  "msg_type": "heartbeat",
  "payload": {
    "agent_id": "...",
    "version": "...",
    "auto_update": true
  }
}
```

**后端流程：**
1. `migrations.sql` 新增迁移：`ALTER TABLE agents ADD COLUMN config_json TEXT NOT NULL DEFAULT '{}'`
2. `config.rs` 新增 `auto_update: bool` 字段（serde default = true）
3. `agent.rs` 新增 `get_agent_config` 和 `update_agent_config` 函数
4. `routes.rs` 注册路由 `GET /api/agents/:agent_id/config` 和 `PATCH /api/agents/:agent_id/config`
5. Hub `ws.rs` heartbeat 处理中解析 `auto_update` 字段，存入 agents 表 `config_json`
