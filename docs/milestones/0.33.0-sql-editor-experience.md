## 子任务详细设计

### 1 SQL 结果分页

- **功能目标**：在 SQL 结果表格底部添加分页控件，支持翻页浏览大量结果
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/sql/SqlResults.vue`（分页 UI + 逻辑）
  - 修改：`packages/rex-console-web/src/i18n/zh.ts`（分页 i18n）
  - 修改：`packages/rex-console-web/src/i18n/en.ts`
- **接口设计**：无 API 改动，纯前端（客户端分页）
- **交互设计**：
  - 结果区底部显示分页控件：`< 1 2 3 ... 10 >` + 每页行数选择器（50/100/500）
  - 当结果行数 ≤ 50 时不显示分页
  - 默认每页 50 行
- **后端流程**：无
- **测试标准**：
  - 结果行数 > 50 时显示分页控件
  - 点击页码可切换页面
  - 每页行数选择器可切换
  - 结果行数 ≤ 50 时不显示分页
- **提交信息**：`feat: add SQL result pagination`

### 2 结果列头点击排序

- **功能目标**：点击结果表格列头可按该列排序，再次点击反转排序
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/sql/SqlResults.vue`（列头点击事件 + 排序图标）
- **接口设计**：无 API 改动，纯前端
- **交互设计**：
  - 列头可点击，鼠标变为 pointer
  - 点击后显示排序图标（↑ 升序、↓ 降序）
  - 再次点击反转排序
  - 第三次点击取消排序
- **后端流程**：无
- **测试标准**：
  - 列头可点击
  - 排序图标正确显示
  - 排序结果正确
  - 取消排序后恢复原顺序
- **提交信息**：`feat: add SQL results column header sort`

### 3 Ctrl+Shift+F 格式化快捷键

- **功能目标**：在 SQL 编辑器中按下 Ctrl+Shift+F 可格式化 SQL
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/sql/SqlEditor.vue`（键盘事件处理）
  - 修改：`packages/rex-console-web/src/i18n/zh.ts`（快捷键提示）
  - 修改：`packages/rex-console-web/src/i18n/en.ts`
- **接口设计**：无 API 改动，纯前端
- **交互设计**：
  - Ctrl+Shift+F / Cmd+Shift+F → 格式化 SQL
  - 工具栏显示快捷键提示：`Ctrl+Shift+F 格式化`
- **后端流程**：无
- **测试标准**：
  - 按 Ctrl+Shift+F 可格式化 SQL
  - 格式化结果与右键菜单"格式化 SQL"一致
- **提交信息**：`feat: add Ctrl+Shift+F format shortcut`

### 4 查询标签"另存为"功能

- **功能目标**：在查询标签右键菜单中添加"另存为…"选项，将当前查询保存为新文件
- **文件结构**：
  - 修改：`packages/rex-console-web/src/features/sql/SqlTabs.vue`（右键菜单）
  - 修改：`packages/rex-console-web/src/features/sql/useSqlTabActions.ts`（另存为逻辑）
  - 修改：`packages/rex-console-web/src/i18n/zh.ts`（另存为 i18n）
  - 修改：`packages/rex-console-web/src/i18n/en.ts`
- **接口设计**：无 API 改动，复用现有 `saveQuery` API
- **交互设计**：
  - 右键菜单项"另存为…"（仅已保存的标签可用）
  - 点击后弹出输入框，输入新文件名
  - 保存成功后切换到新标签
- **后端流程**：无
- **测试标准**：
  - 已保存标签右键菜单显示"另存为…"
  - 点击后弹出输入框
  - 保存成功后切换到新标签
  - 未保存标签不显示"另存为…"
- **提交信息**：`feat: add SQL query tab save as`
