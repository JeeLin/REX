# 0.36.0: 设置页版本总览补全

## Context

0.35.0 完成了 SQL 执行计划功能。PRODUCT.md 要求设置页「更新」区块内包含可折叠的版本总览，显示 Hub 和 Agent 的详细版本信息。当前实现已有 Agent 版本列表基础，但缺少折叠控制、Hub 行图标、平台标签等设计要求的细节。

版本类型：patch（功能补全，无新功能）

## 产品边界

**做什么：**
- 版本总览区块添加折叠/展开控件（默认折叠）
- Hub 行添加图标和「✓ 最新」标记
- Agent 行添加平台标签（如 `linux · arm64`）
- 优化版本总览整体样式

**不做什么：**
- 修改更新检测/下载/应用流程
- 新增后端 API
- 添加新的设置项

## 子任务详细设计

### 1 版本总览折叠/展开控件

- **功能目标**：版本总览区块默认折叠，点击展开显示详细信息
- **文件结构**：修改 `packages/rex-console-web/src/features/settings/UpdateSection.vue`
- **接口设计**：无 API 改动，纯前端
- **交互设计**：
  - 折叠状态：显示「版本总览 ▸」按钮
  - 展开状态：显示「版本总览 ▾」按钮 + 版本信息列表
  - 使用 localStorage 持久化折叠状态
- **测试标准**：TypeScript 类型检查通过，折叠切换正常
- **提交信息**：`feat: add collapsible version overview section`

### 2 Hub 行 + Agent 行增强

- **功能目标**：按 PRODUCT.md 设计增强版本信息展示
- **文件结构**：修改 `packages/rex-console-web/src/features/settings/UpdateSection.vue`
- **接口设计**：
  - 后端已返回 `platform` 字段（如 `linux arm64`），前端需展示
- **交互设计**：
  - Hub 行：⊞ 图标 + `REX Hub` + 版本号 + 「✓ 最新」标记
  - Agent 行：⬡ 图标 + Agent 名称 + 平台标签（`os · arch`）+ 版本号 + 状态
  - 离线 Agent 显示「离线」替代版本号
- **后端流程**：无（API 已返回平台信息）
- **测试标准**：TypeScript 类型检查通过，样式符合设计
- **提交信息**：`feat: enhance version overview with icons and platform tags`
