# 0.24.0: Bug 修复与体验改进

## Context

0.23.1 完成了 CI 构建速度优化。用户在实际使用中反馈了多项体验问题，涵盖后端 API、前端交互、移动端适配等。本里程碑聚焦修复这些问题并完善未实现的设置功能，提升整体产品质量。

版本类型：patch（修复与改进），版本号 0.24.0。

## 产品边界

**做什么：**
- 修复已知后端 API 与数据存储问题（审计日志时间、Agent 下载认证、HTTPS 证书、资源编辑路由）
- 修复前端交互问题（SSH 终端、Redis 交互、侧边栏更新、收藏/最近使用、布局切换拖拽）
- 修复移动端问题（SSH 历史、SFTP 目录导航）
- 完成 SQL 编辑页改造（接近 Navicat 交互）
- 完成设置页未实现功能
- 清理编译/类型警告
- 补充 CHANGELOG

**不做什么：**
- 不引入新协议
- 不修改产品定位
- 不重构核心架构
- 不新增里程碑文档

## 子任务详细设计

### 1 修复审计日志 API 为空

- **功能目标**：修复 `/api/audit-log` 在带 `from/to` 时返回空的问题，确保时间过滤正确。
- **文件结构**：
  - 修改：`crates/rex-hub/src/helpers.rs`（`now_iso` 改为 ISO 8601）
  - 修改：`crates/rex-hub/src/audit.rs`（`today_prefix` 改为 ISO 前缀，测试更新）
  - 修改：`crates/rex-hub/src/helpers.rs`（测试更新）
- **接口设计**：
  - GET `/api/audit-log`：from/to 参数按 ISO 8601 字符串比较
- **后端流程**：写入使用 `chrono::Utc::now().to_rfc3339()` 或类似格式；查询与存储格式一致。
- **测试标准**：单元测试通过；手动验证带时间范围查询返回正确记录。
- **提交信息**：`fix: use iso8601 for audit log time`
