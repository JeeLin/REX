# 0.24.0: 代码质量清理

## Context

0.23.1 完成了 CI 构建速度优化。0.22.0 引入了 Agent 日志查看、远程重启等功能，0.21.0 打通了 Agent 自动更新流程。这些功能实现过程中积累了一些编译警告、未使用的代码和风格不一致。本里程碑进行代码质量清理，不引入新功能。

版本类型：patch（纯修复/重构），版本号 0.24.0。

## 产品边界

**做什么：**
- 修复 Rust 编译警告（dead_code、unused_imports 等）
- 清理前端 TypeScript/ESLint warnings
- 统一代码风格（格式化、命名规范）
- 补充关键路径的单元测试

**不做什么：**
- 不新增功能
- 不修改产品行为
- 不重构架构
- 不修改里程碑文档和开发文档

## 子任务详细设计

### 1 清理 Rust 编译警告

- **功能目标**：修复 `cargo clippy` 和 `cargo check` 产生的所有 warning，使编译零警告。
- **文件结构**：涉及所有 `crates/` 下有警告的源文件。
- **已知警告**：
  - `crates/rex-hub/src/bin/rex-hub.rs:206` — `fn extract_port` dead_code
- **检查标准**：`cargo clippy --workspace --all-targets 2>&1 | grep -E "warning|error"` 无输出。
- **提交信息**：`fix: clean up rust compilation warnings`

### 2 清理前端 lint/type 警告

- **功能目标**：修复前端 `type-check` 和 `lint` 产生的所有 warning。
- **文件结构**：涉及 `packages/rex-console-web/src/` 下的 Vue/TS 文件。
- **检查标准**：`bun run type-check` 和 `bun run lint` 无 error（warning 可忽略，但清理明显的）。
- **提交信息**：`fix: clean up frontend type and lint warnings`
