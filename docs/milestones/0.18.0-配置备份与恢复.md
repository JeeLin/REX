# 0.18.0: 配置备份与恢复

## Context

0.17.0 完成了全局查询功能（后端并行查询 API + 前端模态弹窗）。REX Hub 作为自托管平台，用户的所有环境、资源、配置都存储在本地 SQLite 数据库中。目前缺少配置备份和恢复能力，一旦数据损坏或服务器迁移，用户需要手动重建所有配置。本里程碑提供配置导出/导入功能，支持加密备份和灾难恢复。

版本类型：minor（新功能）

## 产品边界

**做什么：**
- 后端：配置导出 API，将环境、资源、设置导出为加密 JSON 文件
- 后端：配置导入 API，解析备份文件并合并到当前数据库
- 前端：设置页备份面板 UI（导出按钮、导入上传、导入预览）
- 支持可选密码加密备份文件
- 导入时预览差异，支持跳过/覆盖/合并策略
- 支持选择性导出（按环境筛选）

**不做什么：**
- 不做自动定时备份（用户手动触发或自行配置 cron）
- 不做远程存储备份（导出到本地文件）
- 不备份审计日志（审计日志是运行记录，非配置）
- 不备份 Agent 运行时状态（Agent 重连后自动上报）
- 不做跨版本迁移（Hub 和 Agent 版本必须一致）
- 不做数据库迁移/升级（由 migrations.sql 自动处理）

## 子任务详细设计

### 1 配置导出后端 API

**功能目标：**
实现 POST /api/backup/export 端点，将当前 Hub 的配置数据导出为 JSON 文件。支持可选密码加密和环境筛选。加密使用 PBKDF2-SHA256 派生密钥 + AES-256-GCM 对称加密。

**文件结构：**
```text
crates/rex-hub/src/
├── backup.rs           新增：备份导出/导入逻辑
├── db.rs               修改：添加批量查询方法
├── routes.rs           修改：注册备份路由
```

**数据模型：**
```rust
// 备份文件格式
struct BackupFile {
    version: String,           // 备份格式版本 "1.0"
    created_at: String,        // ISO 8601 时间戳
    hub_version: String,       // Hub 版本号 "0.18.0"
    encrypted: bool,           // 是否加密
    salt: Option<String>,      // 加密盐值（Base64）
    nonce: Option<String>,     // AES-GCM nonce（Base64）
    data: String,              // 明文 JSON 或加密后的 Base64 密文
}

struct BackupData {
    environments: Vec<Environment>,
    resources: Vec<Resource>,
    settings: Vec<Setting>,    // settings 表的所有键值对
}
```

**API 设计：**
```text
GET /api/backup/export?env_ids=env1,env2
Query:
  env_ids: 可选，逗号分隔的环境 ID，不传则导出全部
Response: application/octet-stream
  Content-Disposition: attachment; filename="rex-backup-20260625.json"
  Body: BackupFile JSON
```

**后端流程：**
1. 查询所有 environments（或按 env_ids 筛选）
2. 查询这些环境下的所有 resources
3. 查询 settings 表所有配置
4. 组装 BackupFile 结构
5. 返回 JSON 文件（Content-Disposition 触发下载）

**加密流程：** 用户提供 password 时，使用 PBKDF2-SHA256 派生密钥 + AES-256-GCM 加密 data 字段，salt 和 nonce 存储在 BackupFile 中。不加密时 data 字段直接存放明文 JSON。

**测试标准：**
- 导出所有配置，JSON 格式正确
- 按环境 ID 筛选导出，只包含指定环境的资源
- 导出的 JSON 包含 version、created_at、hub_version 字段
- settings 表内容完整导出

### 2 配置导入后端 API

**功能目标：**
实现 POST /api/backup/import 端点，解析备份文件并导入到当前数据库。支持密码解密、冲突检测和合并策略。

**文件结构：**
```text
crates/rex-hub/src/
├── backup.rs           修改：添加导入逻辑
├── db.rs               修改：添加批量插入/更新方法
├── routes.rs           修改：注册导入路由
```

**API 设计：**
```text
POST /api/backup/import
Content-Type: multipart/form-data
Body:
  file: 备份文件（JSON）
  password: 可选，解密密码
  strategy: 合并策略 "skip_existing" | "overwrite"
Response:
  {
    "environments": { "created": 2, "skipped": 1, "updated": 0 },
    "resources": { "created": 5, "skipped": 2, "updated": 0 },
    "settings": { "created": 0, "skipped": 0, "updated": 3 },
    "warnings": ["Environment '阿里云' already exists, skipped"]
  }
```

**合并策略：**
- `skip_existing`（默认）：已存在的跳过，不存在的创建
- `overwrite`：已存在的用备份数据覆盖（保留原 ID）

**冲突检测：**
- 环境：按 ID 判断是否已存在
- 资源：按 ID 判断是否已存在
- 设置：按 key 判断是否已存在

**后端流程：**
1. 接收上传文件，解析 JSON
2. 验证备份格式版本
3. 如果 encrypted=true，用提供的 password 解密
4. 根据 strategy 执行导入：
   - 对每个 environment：检查 ID 是否存在 → 创建/跳过/更新
   - 对每个 resource：检查 ID 是否存在 → 创建/跳过/更新
   - 对每个 setting：检查 key 是否存在 → 创建/跳过/更新
5. 返回导入统计和警告信息

**加密/解密流程：**
```text
加密：
  password + random salt (16 bytes)
    ↓ PBKDF2-SHA256 (100000 iterations)
    ↓ derive 32-byte key
    ↓ AES-256-GCM encrypt(data_json)
    ↓ { salt: base64, nonce: base64, ciphertext: base64 }

解密：
  password + salt (from backup file)
    ↓ PBKDF2-SHA256 (100000 iterations)
    ↓ derive 32-byte key
    ↓ AES-256-GCM decrypt(ciphertext, nonce)
    ↓ data_json
```

**测试标准：**
- 导入新配置成功（created 计数正确）
- 跳过已存在配置（skipped 计数正确）
- 覆盖已存在配置（updated 计数正确）
- 加密文件导入：正确密码解密成功
- 加密文件导入：错误密码返回错误
- 损坏的备份文件返回格式错误提示
- 备份版本不兼容时返回警告


**API 设计：**
```text
POST /api/backup/preview
Content-Type: multipart/form-data
Body:
  file: 备份文件（JSON）
  password: 可选，解密密码
Response:
  {
    "hub_version": "0.17.0",
    "created_at": "2026-06-25T10:00:00Z",
    "encrypted": true,
    "environments": [
      { "id": "env_1", "name": "阿里云", "exists": false },
      { "id": "env_2", "name": "家庭NAS", "exists": true }
    ],
    "resources": [
      { "id": "res_1", "name": "云服务器", "protocol": "ssh", "exists": false },
      { "id": "res_2", "name": "NAS文件", "protocol": "sftp", "exists": true }

    "settings_count": 5
  }
```

**交互设计：**
1. 用户上传文件后，先调用 preview API
2. 弹窗显示预览内容：
   - 备份来源（Hub 版本、备份时间）

   - 资源列表：新建/已存在 标记

3. 用户确认后执行导入
4. 导入完成后显示结果

**测试标准：**
- 预览正确显示备份内容
- 已存在的环境/资源标记为"已存在"
- 加密文件需要先输入密码才能预览
- 预览弹窗可取消关闭
