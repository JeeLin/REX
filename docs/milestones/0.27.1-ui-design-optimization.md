# 0.27.1: UI 设计优化与无障碍

## Context

0.27.0 完成了 UI 一致性与交互反馈（统一三态组件、Toast、ConfirmDialog、表格排序、移动端适配）。本里程碑进一步打磨 UI 细节——激活已搭建但未使用的 Toast 系统，统一手写弹窗为标准组件，完善 i18n 覆盖，增强无障碍支持，清理多主题下的硬编码颜色，规范化操作逻辑。

版本类型：minor（新功能，向后兼容），版本号 0.27.1。

## 产品边界

**做什么：**
- 激活 Toast 反馈：所有 CRUD 操作成功/失败时显示 Toast
- 统一确认弹窗：将手写 modal 替换为 ConfirmDialog 组件
- i18n 补全：消除硬编码中文，所有用户可见文本使用 i18n key
- 无障碍增强：ARIA 标签、键盘导航、焦点管理
- 主题一致性：清理组件内硬编码颜色，统一使用 CSS 变量
- 操作逻辑规范化：错误处理模式统一

**不做什么：**
- 不新增后端 API
- 不新增页面或功能模块
- 不改变路由结构
- 不引入新的设计语言或框架

## 子任务详细设计

### 1 激活 Toast 反馈 + 统一 ConfirmDialog

- **功能目标**：所有 CRUD 操作成功时显示 success Toast，失败时显示 error Toast；所有删除/危险操作使用 ConfirmDialog 组件。
- **文件结构**：
  - 修改：`src/pages/Dashboard.vue`（加载失败 Toast）
  - 修改：`src/pages/Environments.vue`（创建/删除环境）
  - 修改：`src/pages/EnvironmentDetail.vue`（添加/删除资源）
  - 修改：`src/pages/Agents.vue`（重启 Agent）
  - 修改：`src/pages/Terminal.vue`（断开连接弹窗 → ConfirmDialog）
  - 修改：`src/pages/Files.vue`（删除文件弹窗 → ConfirmDialog）
  - 修改：`src/features/workspace/panels/WorkspaceTerminal.vue`（断开弹窗 → ConfirmDialog）
  - 修改：`src/features/workspace/panels/WorkspaceFiles.vue`（删除文件弹窗 → ConfirmDialog）
  - 修改：`src/pages/Settings.vue`（保存设置成功/失败 Toast）
- **接口设计**：
  - useToast()：`success(msg)` / `error(msg)` / `warning(msg)`
  - ConfirmDialog 组件已有，直接替换手写 modal
- **交互设计**：
  - 操作成功 → 右上角绿色 success Toast，3 秒自动消失
  - 操作失败 → 右上角红色 error Toast，5 秒自动消失
  - 删除操作 → ConfirmDialog 模态弹窗，danger 按钮
  - 断开连接 → ConfirmDialog 确认弹窗
- **测试标准**：所有 CRUD 操作有 Toast 反馈，所有删除/断开操作弹出 ConfirmDialog
- **提交信息**：`feat: activate Toast feedback and unify ConfirmDialog across all pages`

### 2 i18n 补全

- **功能目标**：消除所有硬编码中文，所有用户可见文本使用 i18n key。
- **文件结构**：
  - 修改：`src/i18n/zh.ts`（补充新 key）
  - 修改：`src/i18n/en.ts`（补充英文翻译）
  - 修改：各页面（替换硬编码文本）
- **需替换的硬编码文本**：
  - `'加载仪表盘数据失败'` → `t('dashboard.loadFailed')`
  - `'加载环境失败'` → `t('env.loadFailed')`
  - `'加载环境详情失败'` → `t('env.detailLoadFailed')`
  - `'加载 Agent 列表失败'` → `t('agent.loadFailed')`
  - `'加载中...'` → `t('common.loading')`
  - `'连接中...'` / `'已连接'` / `'已断开'` 等状态文本
  - `'此目录为空'` / `'确认断开连接？'` 等弹窗文本
  - `'admin'` 用户名选项
  - `'REX Hub'` / `'自托管 · 开源'` 版本文本
- **测试标准**：`grep -r '[一-鿿]' src/` 找不到非 i18n 的硬编码中文
- **提交信息**：`fix: replace hardcoded Chinese strings with i18n keys`

### 3 无障碍增强

- **功能目标**：核心交互组件添加 ARIA 标签，支持键盘导航，管理焦点。
- **文件结构**：
  - 修改：`src/components/ConfirmDialog.vue`（ARIA role、焦点陷阱）
  - 修改：`src/components/ToastProvider.vue`（ARIA live region）
  - 修改：`src/components/ContextMenu.vue`（键盘导航、ARIA）
  - 修改：`src/components/CommandPalette.vue`（ARIA combobox）
  - 修改：`src/layouts/AppLayout.vue`（导航 landmark、skip link）
  - 修改：`src/pages/Login.vue`（form ARIA）
- **接口设计**：
  - ConfirmDialog：`role="alertdialog"`，`aria-modal="true"`，焦点陷阱（Tab 循环）
  - ToastProvider：`role="status"`，`aria-live="polite"`
  - ContextMenu：`role="menu"`，上下键导航，Esc 关闭
  - AppLayout：`<nav aria-label="main">`，`<main>` landmark，skip-to-content link
- **交互设计**：
  - 打开 ConfirmDialog 时焦点移到确认按钮，关闭时焦点回到触发元素
  - ContextMenu 打开时上下键选择，Enter 执行，Esc 关闭
  - Esc 键关闭所有浮层（ConfirmDialog、ContextMenu、CommandPalette）
- **测试标准**：键盘可完成主要操作流程（登录 → 导航 → 操作 → 确认 → 反馈）
- **提交信息**：`feat: add ARIA labels, keyboard navigation, and focus management`

### 4 主题一致性

- **功能目标**：清理组件内硬编码颜色值，统一使用 CSS 变量。
- **文件结构**：
  - 修改：各组件（替换硬编码颜色为 CSS 变量）
- **需清理的硬编码颜色**：
  - `color: #3FB950` → `color: var(--success)`
  - `color: #F85149` → `color: var(--danger)`
  - `color: #58A6FF` → `color: var(--info)`
  - `color: #D29922` → `color: var(--warning)`
  - `background: rgba(...)` → 使用 CSS 变量或主题感知的值
  - `border-color: #30363D` → `border-color: var(--border)`
- **测试标准**：切换 dark/light/system 主题后所有页面视觉正确
- **提交信息**：`refactor: replace hardcoded colors with CSS variables for theme consistency`

### 5 操作逻辑规范化

- **功能目标**：统一错误处理模式——API 操作失败必须有用户反馈（Toast 或 ErrorState），不静默吞错。
- **文件结构**：
  - 修改：各页面的 catch 块（从静默改为 Toast 反馈）
- **规范**：
  - 页面加载失败 → ErrorState 组件 + retry
  - CRUD 操作失败 → error Toast
  - WebSocket 断连 → 状态栏显示 + 自动重连提示
  - 不允许空 catch 块（除非有 UI 状态反馈）
- **测试标准**：所有 catch 块都有用户可见的反馈机制
- **提交信息**：`fix: add user feedback for all error catch blocks`
