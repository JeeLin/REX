# 0.23.1: CI 构建速度优化

## Context

0.23.0 修复了非 SSH 资源连接。当前 CI 流程（`ci.yml`）中 `docker-agent` 和 `build-hub` 各需约 30 分钟，严重影响开发迭代效率。根因是 `docker-agent` 在 Docker 内用 QEMU 模拟 arm64 从源码编译 Rust，且 `build-hub` 保留了不必要的多平台矩阵。

版本类型：patch（重构/优化），版本号 0.23.1。

## 产品边界

**做什么：**
- `Dockerfile.agent` 改为复制预构建二进制（对齐 `Dockerfile.hub` 模式）
- `docker-agent` job 改为下载 `build-agent` 产物 + 简单打包镜像
- `build-hub` 去掉不必要的多平台矩阵（Hub 只需 Docker 部署，linux-amd64 即可）
- `build-agent` 去掉 Windows 平台（Agent 不支持 Windows Docker）

**不做什么：**
- 不修改 `Dockerfile.hub`（已使用预构建模式）
- 不修改 `check-rust` / `check-web` 质量门禁
- 不修改 Release 发布流程（保留全平台二进制）
- 不修改后端代码

## 子任务详细设计

### 1 Dockerfile.agent 改为预构建模式

**功能目标：**
将 `Dockerfile.agent` 从多阶段源码编译改为复制预构建二进制，对齐 `Dockerfile.hub` 的模式。

**文件结构：**
```text
Dockerfile.agent   修改：去掉 builder stage，改为 COPY 预构建二进制
```

**当前 Dockerfile.agent（多阶段编译）：**
```dockerfile
FROM rust:1.87-slim AS builder
RUN apt-get update && apt-get install -y pkg-config libssl-dev && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY . .
RUN cargo build --release -p rex-agent

FROM debian:bookworm-slim
# ... 安装依赖、创建用户、COPY --from=builder
```

**改为（预构建模式，对齐 Dockerfile.hub）：**
```dockerfile
FROM debian:bookworm-slim
RUN apt-get update && \
    apt-get install -y --no-install-recommends ca-certificates && \
    rm -rf /var/lib/apt/lists/*
RUN useradd -r -s /bin/false agent
WORKDIR /app
RUN mkdir -p /app/data && chown agent:agent /app/data
COPY dist/rex-agent /app/rex-agent
RUN chmod +x /app/rex-agent
USER agent
ENV DATA_DIR=/app/data
CMD ["/app/rex-agent", "--worker"]
```

**提交信息：**
```
ci: switch Dockerfile.agent to pre-built binary mode
```

### 2 CI workflow 优化

**功能目标：**
修改 `.github/workflows/ci.yml`：
1. `docker-agent` job 改为下载 `build-agent` 产物 + 打包镜像（不再从源码编译）
2. `build-hub` 去掉非 linux-amd64 平台（Hub 只需 Docker 部署）
3. `build-agent` 去掉 Windows 平台（Agent 不支持 Windows Docker）

**文件结构：**
```text
.github/workflows/ci.yml   修改：优化 job 依赖和矩阵
```

**关键改动：**

`build-hub` 矩阵精简：
```yaml
# 之前：5 个平台
matrix:
  include:
    - target: x86_64-unknown-linux-gnu    # 保留（Docker + 二进制下载）
    - target: aarch64-unknown-linux-gnu   # 保留（Docker arm64 + 二进制下载）
    - target: x86_64-apple-darwin         # 保留（二进制下载）
    - target: aarch64-apple-darwin        # 保留（二进制下载）
    - target: x86_64-pc-windows-msvc      # 保留（二进制下载）
```

> 实际上 build-hub 的多平台是给 Release 二进制下载用的，不应该去掉。真正需要优化的是 docker-agent。

`docker-agent` 改为使用预构建产物：
```yaml
docker-agent:
  needs: [build-agent]  # 改为等待 build-agent 完成
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v7
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v4
    - name: Log in to GHCR
      uses: docker/login-action@v4
      # ...
    - name: Download agent binary (linux-amd64)
      uses: actions/download-artifact@v8
      with:
        name: agent-linux-amd64
        path: dist/
    - name: Download agent binary (linux-arm64)
      uses: actions/download-artifact@v8
      with:
        name: agent-linux-arm64
        path: dist/arm64/
    - name: Prepare agent README
      run: cp docs/agent-readme.md README.md
    - name: Build and push Agent image
      uses: docker/build-push-action@v7
      with:
        context: .
        file: Dockerfile.agent
        platforms: linux/amd64,linux/arm64
        push: true
        # ...
```

> 注意：Dockerfile.agent 需要支持多架构 COPY（通过 buildx 的 `TARGETARCH` 环境变量），或者为每个架构单独构建后合并 manifest。

**简化方案（推荐）：**
由于 `build-agent` 已经构建了 linux-amd64 和 linux-arm64，`docker-agent` 可以：
- 下载两个架构的二进制
- 为每个架构分别构建单架构镜像
- 用 `docker buildx imagetools create` 合并为多架构 manifest

**提交信息：**
```
ci: optimize docker-agent to use pre-built binaries
```
