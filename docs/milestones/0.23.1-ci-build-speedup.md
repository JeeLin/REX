# 0.23.1: CI 构建速度优化

## Context

0.23.0 修复了非 SSH 资源连接。CI 流程中 `docker-agent` 和 `build-hub` 各需约 30 分钟，严重影响开发迭代效率。`docker-agent` 使用 QEMU 在 Docker 内模拟 arm64 从源码编译 Rust（~25-30min），而 `build-agent` 已经原生编译了所有平台二进制。`Dockerfile.agent` 是多阶段编译，未利用预构建产物。

版本类型：patch（CI 优化），版本号 0.23.1。

## 产品边界

**做什么：**
- `Dockerfile.agent` 改为复制预构建二进制（对齐 `Dockerfile.hub`）
- `docker-agent` job 改为下载预构建产物 + 打包镜像（消除 QEMU 编译）
- `build-hub` 保持不变（Release 流程需要全平台二进制用于 zip + changelog 发布）

**不做什么：**
- 不修改 `Dockerfile.hub`（已使用预构建模式）
- 不修改质量门禁（check-rust / check-web）
- 不修改 Release 发布流程
- 不修改后端代码

## 子任务详细设计

### 1 Dockerfile.agent 改为预构建模式

**功能目标：**
将 `Dockerfile.agent` 从多阶段源码编译改为复制预构建二进制。使用 Docker BuildKit 的 `TARGETARCH` 变量选择对应架构的二进制。

**文件结构：**
```text
Dockerfile.agent   修改：去掉 builder stage，改为 COPY dist/ 预构建二进制
```

**新 Dockerfile.agent：**
```dockerfile
# Agent Docker 镜像：复制 CI 预构建的二进制
#
# CI 流程：
# 1. build-agent → dist/rex-agent (amd64), dist/arm64/rex-agent (arm64)
# 2. 本 Dockerfile 根据 TARGETARCH 选择对应二进制

FROM debian:bookworm-slim

RUN apt-get update && \
    apt-get install -y --no-install-recommends ca-certificates && \
    rm -rf /var/lib/apt/lists/*

RUN useradd -r -s /bin/false agent

WORKDIR /app
RUN mkdir -p /app/data && chown agent:agent /app/data

# TARGETARCH 由 docker buildx --platform 自动设置（amd64/arm64）
COPY dist/rex-agent /tmp/rex-agent-amd64
COPY dist/arm64/rex-agent /tmp/rex-agent-arm64
RUN if [ "$(uname -m)" = "aarch64" ]; then \
      cp /tmp/rex-agent-arm64 /app/rex-agent; \
    else \
      cp /tmp/rex-agent-amd64 /app/rex-agent; \
    fi && \
    rm -f /tmp/rex-agent-amd64 /tmp/rex-agent-arm64 && \
    chmod +x /app/rex-agent

USER agent
ENV DATA_DIR=/app/data

CMD ["/app/rex-agent", "--worker"]
```

> `uname -m` 在 buildx `--platform` 下返回目标架构（不是构建机器架构），所以可以用它选择二进制。

**提交信息：**
```
ci: switch Dockerfile.agent to pre-built binary mode
```

### 2 CI workflow 优化

**功能目标：**
修改 `.github/workflows/ci.yml`：

1. `docker-agent` job 改为下载 `build-agent` 产物 + 打包镜像（消除 QEMU 源码编译）
2. `build-hub` 去掉非 Docker 必需平台（Hub 只需 linux-amd64 Docker 镜像，Release 二进制保留）

**文件结构：**
```text
.github/workflows/ci.yml   修改：优化 job 依赖和矩阵
```

**`docker-agent` 改动：**

```yaml
docker-agent:
  needs: [build-agent]  # 改为等待 build-agent 完成（不再依赖 check-rust/check-web）
  runs-on: ubuntu-latest
  permissions:
    contents: read
    packages: write
  steps:
    - uses: actions/checkout@v7
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v4
    - name: Log in to GHCR
      uses: docker/login-action@v4
      with:
        registry: ${{ env.REGISTRY }}
        username: ${{ github.actor }}
        password: ${{ secrets.GITHUB_TOKEN }}
    - name: Download agent binary (linux-amd64)
      uses: actions/download-artifact@v8
      with:
        name: agent-linux-amd64
        path: dist/
    - name: Download agent binary (linux-arm64)
      uses: actions/download-artifact@v8
      with:
        name: agent-linux-arm64
        path: dist/arm64/
    - name: Prepare agent README
      run: cp docs/agent-readme.md README.md
    - name: Docker meta
      id: meta-agent
      uses: docker/metadata-action@v6
      with:
        images: ${{ env.AGENT_IMAGE }}
        tags: type=ref,event=tag
    - name: Build and push Agent image
      uses: docker/build-push-action@v7
      with:
        context: .
        file: Dockerfile.agent
        platforms: linux/amd64,linux/arm64
        push: true
        tags: ${{ steps.meta-agent.outputs.tags }}
        labels: ${{ steps.meta-agent.outputs.labels }}
        cache-from: type=gha
        cache-to: type=gha,mode=max
```

**关键变化：**
- 去掉 QEMU setup（不再需要模拟 arm64 编译）
- 去掉 `check-rust` / `check-web` 依赖（改为依赖 `build-agent`）
- 下载两个架构的预构建二进制到 `dist/` 和 `dist/arm64/`

**`build-hub` 保持不变**（Release 二进制需要多平台，Docker 镜像只需 linux-amd64 已由 `docker-hub` 处理）。

**预估效果：**
- `docker-agent`：~30min → ~2-3min（只需下载二进制 + 打包镜像）
- 总 CI 时间从 ~35min 降到 ~12min（瓶颈变为 `build-agent` 的并行编译）

**提交信息：**
```
ci: optimize docker-agent to use pre-built binaries
```
