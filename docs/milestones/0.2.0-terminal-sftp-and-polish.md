## 子任务详细设计

### 1 终端内置 SFTP 面板

**功能目标：**
SSH 终端工具栏点击「📁 SFTP」按钮，在右侧打开内嵌 SFTP 文件管理面板（split view），共享同一个 SSH 连接。面板可折叠/展开，支持文件浏览、上传、下载。

**文件结构：**
```text
packages/rex-console-web/src/
├── features/terminal/
│   ├── TerminalSftp.vue         新增：SFTP 面板组件
│   └── useTerminalSftp.ts       新增：SFTP WebSocket 管理 composable
├── pages/Terminal.vue           修改：添加 SFTP 面板集成
├── api/terminal.ts              修改：添加 SFTP 相关 API
├── i18n/zh.ts, en.ts            修改：添加 SFTP 相关 i18n 键
```

**接口设计：**
SFTP 面板复用现有文件管理 REST API（`crates/rex-hub/src/files.rs`）：
```text
GET    /api/resources/:resource_id/files?path=/              # 列出目录
GET    /api/resources/:resource_id/files/download?path=      # 下载文件
POST   /api/resources/:resource_id/files/upload?path=        # 上传文件
POST   /api/resources/:resource_id/files/mkdir               # 创建目录
POST   /api/resources/:resource_id/files/touch               # 创建文件
DELETE /api/resources/:resource_id/files?path=               # 删除文件
PUT    /api/resources/:resource_id/files/rename              # 重命名
```

TerminalSftp.vue 组件设计：
```vue
<template>
  <div class="terminal-sftp" v-if="visible">
    <div class="sftp-toolbar">
      <span>📁 SFTP</span>
      <input class="sftp-path" v-model="currentPath" @keydown.enter="loadDir" />
      <button @click="toggle">✕</button>
    </div>
    <div class="sftp-list">
      <div class="sftp-item" v-for="item in items" :key="item.name"
           @dblclick="navigate(item)" @contextmenu.prevent="showCtxMenu($event, item)">
        <span>{{ item.is_dir ? '📁' : '📄' }}</span>
        <span>{{ item.name }}</span>
        <span>{{ item.is_dir ? '' : formatSize(item.size) }}</span>
      </div>
    </div>
    <div class="sftp-status">
      {{ currentPath }} · {{ items.length }} items
    </div>
  </div>
</template>
```

**交互设计：**
- 工具栏「📁 SFTP」按钮 → 展开/折叠右侧面板
- Split view：终端占左侧 60%，SFTP 面板占右侧 40%
- 面板顶部：路径输入框 + 关闭按钮
- 文件列表：双击文件夹进入，双击 `..` 返回上级
- 右键菜单：下载、复制路径、复制文件名、重命名、删除
- 拖拽文件到终端区域 → 将文件路径粘贴到终端
- 面板底部状态栏：当前路径 + 文件数

**后端流程：**
复用现有 `rex-ssh` crate 的 SFTP 能力。终端已通过 SSH WebSocket 连接到后端，SFTP 面板通过同一 resource_id 的 SFTP API 操作。后端无需新实现，只需前端对接已有 API。

**测试标准：**
- SFTP 按钮展开/折叠面板
- 目录浏览正常（进入/返回上级）
- 文件列表显示名称、大小
- 右键菜单下载/复制路径/删除
- 拖拽文件路径到终端
- 面板尺寸可调整（可选）

**提交信息：**
```

```

### 3 Settings 个人信息区块

**功能目标：**
Settings 页面添加「个人信息」区块，支持查看和修改用户名、密码。

**修改文件：**
```text
packages/rex-console-web/src/features/settings/
├── ProfileSection.vue           新增：个人信息区块
├── index.ts                     修改：导出
packages/rex-console-web/src/pages/Settings.vue     修改：添加 ProfileSection
packages/rex-console-web/src/api/settings.ts        新增：用户信息 API
packages/rex-console-web/src/i18n/zh.ts, en.ts      修改：添加 i18n 键
```

**接口设计：**
```text
GET  /api/user/profile              # 获取用户信息
PUT  /api/user/profile              # 更新用户名
PUT  /api/user/password             # 修改密码
```

更新用户名请求：
```json
{ "username": "new_name" }
```

修改密码请求：
```json
{ "current_password": "old", "new_password": "new" }
```

**后端流程：**
Hub 已有用户表（`users`），只需添加 profile CRUD handler。密码修改需验证当前密码。

**组件设计：**
```vue
<!-- ProfileSection.vue -->
<template>
  <SettingsSection>
    <template #header>{{ t('settings.profile.title') }}</template>
    <div class="settings-row">
      <div class="settings-row-info">
        <div class="settings-row-label">{{ t('settings.profile.username') }}</div>
      </div>
      <button class="btn btn-ghost btn-sm" @click="editUsername">{{ t('settings.profile.edit') }}</button>
    </div>
    <div class="settings-row">
      <div class="settings-row-info">
        <div class="settings-row-label">{{ t('settings.profile.password') }}</div>
        <div class="settings-row-desc">{{ t('settings.profile.lastChanged', { time: passwordChangedAt }) }}</div>
      </div>
      <button class="btn btn-ghost btn-sm" @click="editPassword">{{ t('settings.profile.change') }}</button>
    </div>
  </SettingsSection>
</template>
```

**测试标准：**
- 显示当前用户名
- 修改用户名成功
- 修改密码需验证当前密码
- 密码不匹配提示错误

**提交信息：**
```

```

### 4 Dashboard 审计统计 + SQL 消息标签 + 标签关闭改进

**功能目标：**
1. Dashboard「今日操作数」从后端审计日志 API 获取真实数据
2. SQL 结果区添加「消息」标签，展示执行结果消息（affected rows、错误信息）
3. SQL 查询标签右键菜单添加「关闭全部」「关闭已保存的」

**修改文件：**
```text
crates/rex-hub/src/
├── audit.rs                     修改：添加统计 API
├── routes.rs                    修改：注册统计路由
packages/rex-console-web/src/
├── api/audit.ts                 新增：审计统计 API
├── pages/Dashboard.vue          修改：调用统计 API
├── features/sql/SqlResults.vue  修改：添加消息标签

├── i18n/zh.ts, en.ts            修改：添加 i18n 键
```

**接口设计：**
```text
GET /api/audit/stats?period=today   # 今日操作统计
```

响应：
```json
{
  "data": {
    "total": 42,
    "success": 38,
    "failed": 4
  }
}
```

**SQL 消息标签：**
- 结果区 header 添加「消息」tab

- 执行错误时显示：`ERROR: Table 'users' doesn't exist`
- 消息支持复制

**SQL 标签关闭改进：**
- 右键菜单添加「关闭全部」— 关闭所有查询标签
- 右键菜单添加「关闭已保存的」— 关闭所有已保存的查询标签（保留未保存的）

**测试标准：**
- Dashboard 今日操作数显示真实数据
- SQL 消息标签展示执行结果/错误
- 关闭全部标签正常
- 关闭已保存标签只保留未保存的

**提交信息：**
