# 0.20.0: 完成数据连接器实现

## Context

0.19.0 完成了观测与诊断功能，添加了系统指标采集、健康检查 API 和指标聚合查询。虽然 metrics.rs 中添加了 sysinfo 依赖和框架，但实际系统指标仍为模拟数据。同时，0.10.0 和 0.11.0 分别实现了 SQLite 和 S3/MinIO 协议支持，但相应的连接器实现仍包含大量 TODO 注释，尚未完成真实的数据库连接和操作。

本里程碑旨在完成数据连接器的真实实现，使得 SQLite、S3/MinIO 连接器能够正常工作，并将指标中的模拟数据替换为真实的系统监控数据。

## 产品边界

**做什么：**
- 完成 SQLite 连接器的实际数据库连接、查询执行和结果获取
- 完成 S3/MinIO 连接器的实际对象存储操作（列举桶、列举对象、上传、下载等）
- 将 metrics.rs 中的模拟系统指标替换为使用 sysinfo crate 的真实数据
- 确保所有新增功能有充分的单元测试覆盖

**不做什么：**
- 不添加新的协议支持
- 不修改现有工作的 SSH、MySQL、PostgreSQL、Redis、Docker 连接器
- 不进行前端 UI 更改（后端功能为前端团队提供 API 支持）

## 子任务详细设计

### 1 完成 SQLite 连接器实现

**功能目标：**
在 crates/rex-sqlite/src/connector.rs 中实现 SQLiteConnector trait 的所有方法，使其能够：
- 打开 SQLite 数据库文件连接
- 执行 SQL 查询并返回结果
- 获取表结构信息
- 正确关闭数据库连接

**文件结构：**
```text
crates/
├── rex-sqlite/
│   ├── src/
│   │   ├── connector.rs      修改：实现实际的 SQLite 连接和操作
│   │   └── lib.rs
│   └── Cargo.toml
```

**接口设计：**
实现 FileConnector trait（继承自 rex-transfer），包括：
- connect() -> Result<Box<dyn FileSystem>>
- 其中 FileSystem 需要实现：
  - open(path) -> Result<Box<dyn File>>
  - read(file) -> Result<Vec<u8>>
  - write(file, data) -> Result<()>
  - list(path) -> Result<Vec<PathEntry>>
  - metadata(path) -> Result<Metadata>
  - 等等...

**后端流程：**
1. 使用 rusqlite crate 打开数据库连接
2. 实现文件系统抽象以适配 rex-transfer 框架
3. 将 SQL 查询转换为适当的文件系统操作（对于 SQL 查询场景）
4. 确保错误处理和资源清理

**测试标准：**
- 能够成功连接到 SQLite 数据库文件
- 能够执行基本的 SQL 查询（SELECT, INSERT, UPDATE, DELETE）
- 能够获取表结构信息
- 所有操作都有适当的错误处理
- 单元测试覆盖率达到 80%+

**提交信息：**
```
feat: complete sqlite connector implementation
```

### 3 实现真实系统指标采集

**功能目标：**
在 crates/rex-hub/src/metrics.rs 中将 get_health() 函数中的模拟数据替换为使用 sysinfo crate 获取的真实系统指标。

**文件结构：**
```text
crates/
├── rex-hub/
│   ├── src/
│   │   ├── metrics.rs      修改：使用真实系统指标替换模拟数据
│   │   └── lib.rs
│   └── Cargo.toml
```

**实现细节：**
1. 使用 sysinfo crate 获取：
   - CPU 使用率
   - 内存使用率  
   - 磁盘使用率
   - 进程运行时间
2. 保持现有的 Agent 连接统计（这些已经是真实数据）
3. 确保在获取系统信息时不会阻塞事件循环（使用 tokio::task::spawn_blocking 或类似机制）
4. 处理系统信息获取失败的情况（降级为安全默认值或错误状态）

**后端流程：**
1. 在系统启动时初始化 sysinfo 系统对象
2. 定期更新系统指标（或在需要时实时获取）
3. 在 get_health() 中使用真实数据填充响应
4. 保持现有的数据库和连接统计不变

**测试标准：**
- 系统指标能够正确获取并反映实际系统状态
- 在不同操作系统（Linux/macOS/Windows）上都能工作
- 性能开销可接受（不阻塞主要事件循环）
- 单元测试模拟系统信息获取
- 集成测试验证健康 API 返回合理值

**提交信息：**
```
feat: implement real system metrics collection
```
